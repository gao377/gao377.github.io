第十一幕

档案说，“我”来过这里——从原始记录到有边界的机器记忆

全书路线

泵站的受控试转刚刚结束，我在运行记录旁边找到两份彼此矛盾的旧档案：一份说动作已经施加，设备却没有变化；另一份说设备恢复了，却找不到动作真正发生过的证据。它们都声称描述同一次过去。

这一幕先不让任何学习模型接管身体。我从“哪份记录能说明发生过什么”开始，保留原始记录与来路，再把没有边界的连续日志整理成一段段证据强弱不同的经历候选。博士只在问题出现后引入来源、数据合同、谱系、记忆索引、因果图、反事实、评测基准与系统辨识。

当档案能够说明自己缺了什么，我们才让学习方法进入：学习状态估计、可通行性、开放词汇导航、学习增强运动规划、视觉—语言—动作模型和世界模型都只提出影子候选，并始终与传统或混合基线、硬约束、停止路径和当前身体版本配对。最后，我、排水渠巡检设备和城市保障运输设备各自保留本地经验，通过可复核的联邦学习轮次、隐私账本与身体兼容矩阵形成车队候选，而不是一个吞并所有身体的共同策略。

这一幕沿二十五章把档案、学习候选和跨身体经验接成同一条因果链。产物按以下顺序相接：

有来源的经历登记
→ 数据含义与动作事实合同
→ 可追溯的数据集登记
→ 保留冲突的记忆索引
→ 失败问题与证据图谱
→ 彼此隔离的训练、留出与回归集合
→ 可解释的失败调查队列
→ 学习导航影子候选
→ 标明输出对象的学习运动候选
→ 视觉—语言—动作影子候选
→ 可比较的世界模型查询接口
→ 世界模型影子候选
→ 保留多种来源与冲突的共享世界假设
→ 有边界的候选发布包
→ 本地学习的聚合轮次记录
→ 隐私预算与可见信息记录
→ 保留身体差异和回滚路径的车队候选

记录怎样成为一段有来源的经历

  1. 两份记录都说是同一次故障，为什么结果相反
  2. 一段经历从哪里开始，哪些边界只是后来猜的
  3. 记录里的“动作”，为什么不一定真的发生
  4. 图像、作用和接管，怎样属于同一件事

原始记录先保存字节内容、内容指纹、来源与来路；字段更全或复制更多都不能替历史投票。经历整理规则再把 12 段旧片段分为 4 段显式边界、5 段推断候选和 3 段无法切分，后来推断出来的边界不会被改写成原生事实。

动作被拆成提出、接受、实际施加与结果四个阶段；标定、坐标系、单位、责任端和版本共同决定数组在物理世界里的意义。多模态对齐先理顺时钟和事件窗口，接触、人员接管与版本重置的两侧不做插值；轨迹相似也不能替代任务成功判据。

数据很多，为什么仍可能不可用、没有覆盖或找不到来处

  1. 字段都齐全了，为什么这段数据仍不能用
  2. 数据已经很多，为什么关键失败还没有出现
  3. 清洗后的数据，怎样找到它原来的样子
  4. 最相似的记录，为什么未必是同一段过去

完整度、数据结构与物理有效性、时序、跨模态一致性、不确定度、标签来源和具体用途要分开判断，不能压成一个总质量分。覆盖分析先写出身体、任务、设计运行域、人员、通信、失败和尾部情形的联合条件，再数彼此独立的经历组；相邻画面和同源派生不会制造新的独立覆盖。

数据集登记让“原始记录→转换→派生记录→数据集→数据划分→候选”的依赖可以追溯。上游证据一旦撤销，依赖它的后继会被标成过期，而不是悄悄改写成“最新版”。记忆索引先按身份与当前兼容性过滤，再做语义排序；相似度第一名只是候选，矛盾和旧片段的未知身份仍被保留。

反复一起出现，为什么仍不能直接叫作原因或进步

  1. 每次潮湿都失败，潮湿就是原因吗
  2. 如果当时停下来，结果真的会不同吗
  3. 测试分数变好，是不是因为我提前看过答案
  4. 一个分数，怎样容得下不确定和最坏情况

因果图把混杂因素、中介变量、碰撞点和选择机制，与“观察到潮湿时的失败概率”及“主动改变潮湿条件后的失败概率”分开；多个同样符合观察的有向无环图给出不同结论时，根因保持未识别。反事实再区分实际世界、潜在结果、结构因果模型、群体效应和单次经历的另一种结局，模型预测不能替历史保存一条没有发生的路。

当前 40 组彼此独立的经历固定分为 24/8/8 组训练、验证与最终留出记录，另有 12 个独立回归组；同一原始记录、同一段经历、近重复记录、站点与时段以及成功判据造成的泄漏，都要按照评测问题分组隔离。评测基准同时报告分子与分母、缺失/未知/未运行、配对差、区间、尾部结果和零容忍事件；8 组最终留出记录的小样本，不能被漂亮的百分比隐藏。

实验怎样说明模块、参数与下一次调查

  1. 拿掉一个模块后变差，究竟能说明什么
  2. 记录已经很多，为什么有些参数仍然学不出来
  3. 失败太多，下一次先看哪一个

消融实验要冻结输入、预处理、数据、算力与延迟、后备路径、随机种子、构建版本和环境；隐藏变化还没有分开时，只能报告整个组合的差异。四参数系统辨识简化切片的奇异值为 [5.2,1.8,0.21,0.004]，第四个方向很弱；普通最小二乘、加权最小二乘、约束、正则化和贝叶斯后验，都不能创造数据没有激发出来的信息。

失败调查队列不按频次或一个神秘总分自动排序。严重度、复发次数、新颖性、不确定度、可复现性、覆盖空洞、回归价值和补证成本要分开保留；低频严重险情不会被高频小停顿挤掉，未知新类也不会因为难复现而被删除。

学习候选怎样提出动作与未来，而不夺走世界与身体的边界

  1. 传统定位看不清时，学习表示能补什么
  2. 学习地图说能走，传统地图说不能，听谁的
  3. “去旧泵房旁边”究竟指哪里
  4. 视觉和语言，能不能直接决定下一段动作
  5. 同一个动作，为什么会有几种不同的未来
  6. 能预测许多未来，就算理解世界了吗
  7. 大家记得的世界不一样，还能一起行动吗

学习状态、神经地图与可通行性都只是带有来源、不确定度和分布外状态的观测或代价候选；学习得到的软代价不能覆盖几何、接触、权限和停止等硬约束。开放词汇与视觉—语言导航产生指称对象和子目标；若仍有多个会导致不同动作的解释，就先询问说话者。最近的路线也不能穿过未经许可的工作区。

学习运动方法先按输出对象分开：采样分布、搜索启发和碰撞风险仍由显式搜索与精确检查接手；路径或轨迹候选仍经过身体版本、限位、连续碰撞、动力学、接触和时限检查；动作块则进入技能与控制监督器。视觉—语言—动作影子候选使用 1 Hz/500 ms 设计预算；一次候选在 430 ms 返回，仍会因为下游时延未知而不能接受。

世界模型先按“预测什么、怎样使用、规律从哪里来”比较物理、潜在、对象、三维和生成式未来；同一句动作必须带层级、坐标系、单位、时域和身体版本。随后，当前短时潜在模型候选才采用 2 Hz/250 ms 教学预算：滚动推演不断与现实重新核对，误差、模型分歧、超出经验范围或约束冲突扩大时就停止。共享世界保存并列假设、来源、时钟、身份、权限、有效期限与冲突，不用一个“最新值”制造唯一世界，也不从档案裁定“我”的主体身份。

本地经验怎样形成车队候选，又在隐私与身体前停下

  1. 每个人都在本地学习，怎样形成一个共同候选
  2. 不交出原始记录，就一定保护了隐私吗
  3. 同一个策略换到不同身体，什么时候必须回滚

这次联邦平均（FedAvg）用 48/30/12 个本地样本和 +0.08/-0.04/+0.02 三个更新得到加权增量 0.032。这只核对聚合如何计算；各端数据分布不同、关键参与者掉线与某一身体的尾部退化，仍可能使这一轮中止、限域或回滚。

安全聚合（Secure Aggregation）的“三端至少两端”合同，只限制聚合方看见单端更新，并不自动带来差分隐私、身份认证、操作授权或策略安全。差分隐私切片把范数 1.6 的更新按 C=1.0 裁剪，并登记 σ=1.2 的噪声机制；ε 由冻结的隐私会计方法给出，三轮基本组合的上界为 (2.4,3×10^-5)，这不是“绝不会泄露”的承诺。

三端共享的 E-3 编码器与 128×64 参数块只形成待验证的合并候选；人形、排水渠和运输设备的动作输出头分别为 24/8/6 维，必须保持身体专用。平均改善不能覆盖排水渠的尾部退化，旧权重文件也只有在依赖仍有效、运行状态能够恢复时才是真正的回滚路径。候选不取得安全关键控制权。

返回第十幕：每个模块都正确，泵站为什么仍然不工作

三种身体现在带着来源明确的共享世界、本地经验、联邦轮次、隐私账本和车队候选走向同一场暴雨，却仍没有共同目标、任务分配、承诺与拒绝。第十二幕将从“每个人都完成了自己的任务，为什么演练还是失败了”开始：智能让我们各自能够行动，社会性则要求我们学会共同决定、互相设限，并在失败后共同修复。
