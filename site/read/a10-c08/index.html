第十幕

同一个公式换一种实现，还是同一个结果吗——数值等价、内存与执行所有权

为了让控制循环更快，我把一段验证过的参考计算换成新的运行实现。两边都写着同一组公式，输出向量逐项只差最后几位；可一边把约束余量判断为可接受，另一边却进入 fallback。另一次，两边数值完全相同，新实现仍偶尔错过期限——它在循环里临时申请内存，恰好停顿在最不该停顿的地方。

我这才意识到，“算得差不多”与“在系统里等价”并不是同一件事。博士把要核对的范围称作数值等价与执行等价：前者不仅比较浮点数，还比较由数值触发的决定；后者还要说明谁拥有内存、谁能阻塞线程、何时完成以及失败后留下什么。

比较之前，先冻结两边究竟算什么

参考实现与候选实现必须接收同一份不可变输入快照，并明确：

  • dtype、shape、维度顺序与内存布局；
  • 单位、坐标系、角度范围与正负方向；
  • 身体、地图、任务、配置与模型 revision；
  • 缺失值、无穷值和无效四元数的处理；
  • 求解器容差、最大迭代、初值、随机种子和停止条件；
  • 并行归约、指令优化与 fast-math 是否允许改变运算顺序。

若一边接收米、另一边接收毫米，或一边把矩阵按行解释、另一边按列解释，逐项误差公式再漂亮也没有意义。输入契约不同，应先判接口不等价。

绝对容差照顾接近零的量，相对容差照顾尺度较大的量。容差必须在看见候选结果之前，依据传感分辨率、模型误差、控制敏感性和安全余量冻结；不能先运行，再把阈值放宽到恰好通过。

不同量还需要不同的比较方式。角度要先处理 2π2\pi2π 绕回；四元数 qqq 与 −q-q−q 表示同一旋转，不能只逐元素相等；概率要检查归一化与范围；约束残差要保留正负意义。若出现 NaN 或无穷值，默认行为应是显式失败并进入声明的处置，而不是让比较表达式悄悄返回 false 后继续。

系统关心的是决定是否相同

即使式（1）通过，两边仍可能跨过同一条决策边界。例如参考实现给出很小的正余量，候选实现给出很小的负余量；向量范数几乎相同，却分别走向 nominal 与 fallback。因此等价检查至少分四层：

  1. 中间量与最终数值是否在预先冻结的容差内；
  2. 约束可行性、接触模式和有效/无效标志是否一致；
  3. 选中的动作、fallback 与状态机转移是否一致；
  4. reason code、依赖 revision 和证据字段是否一致。

若两边都选择相同动作，也不能立刻判定等价：候选实现可能丢掉约束检查，只是本次碰巧得到相同输出。反过来，两个动作略有差别却都在同一安全等价类内，也要由事先定义的任务判据说明，不能临时凭肉眼决定。

运算顺序会改变最后几位，也可能改变分支

浮点加法不满足严格结合律。并行归约、向量化、融合乘加和不同求解器可能改变运算顺序；相同随机种子也只有在随机数生成器、调用顺序与并行调度一致时才有意义。因此测试要保留构建标识、编译选项、库版本、线程数、种子和输入哈希。

确定性并不等于正确。每次稳定地产生同一个错误结果，只说明错误可复现。参考实现也不是天然真理：它必须来自可追溯的定义，并由独立性质、约束和物理检查共同约束。

数值相同，也可能不是同一种运行行为

控制路径里的内存与执行所有权需要写清：

  • 哪个进程和线程读取输入、写出输出；
  • 缓冲区由谁分配、复用与释放，借用期何时结束；
  • 回调由哪个 executor 调度，是否可能并发重入；
  • 哪些锁、文件、日志或网络操作可能阻塞；
  • 队列是否有界，溢出时由谁决定丢弃或降级。

快循环应尽量预分配工作区，避免不受界的动态分配、页面错误和垃圾回收。共享缓冲区若没有清晰所有权，生产者可能在消费者计算途中覆盖内容；“零拷贝”便从性能优化变成数据竞争。输出对象也要带 generation 或序号，防止旧指针被误当成当前结果。

运行测试至少区分冷启动与稳态、缓存冷与热、单线程与并发干扰。平均耗时相同不能排除一次罕见分配停顿；这一问题将在下一章用最坏执行与期限继续展开。

一条可复核的替换路径

冻结接口、输入快照、容差、决策判据与失败语义
保存参考输出、中间检查、动作与 reason code
在离线固定输入上比较数值和决定
注入 NaN、边界值、角度绕回、revision 变化与求解失败
检查内存所有权、队列上界、阻塞点与并发重入
分别记录冷启动、稳态和受干扰运行
差异未解释前，候选实现不能取得动作所有权

这条路径不会因为候选实现“通常更快”而缩短。性能是替换条件之一，不是抹去语义差异的理由。

练习

练习 1：向量很接近，保护分支却不同

参考实现与候选实现的输出逐项通过容差，但约束余量一正一负，分别选择 nominal 与 fallback。决定是否可以发布候选实现，并安排定位与证据保存。

练习 2：四元数整体变号

两边输出的姿态四元数恰好互为相反数，逐项比较失败。判断这是表示差异还是物理差异，还要检查什么才能下结论。

练习 3：结果完全相同，却偶尔超时

候选实现每隔一段时间在快环里重新分配缓冲区，输出始终与参考一致，但一次停顿越过控制期限。给出系统处置和实现修正，说明为什么数值回归不足以放行。

参考答案

数值和所有权都说清以后，我仍只知道一些运行“通常很快”。最快的一次、平均的一次，都回答不了我真正担心的问题：最慢的一次会不会赶不上身体？
