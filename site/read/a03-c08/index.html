第三幕

穿过通道以后，哪些地方仍不能说安全——一次四足行走怎样留下可复核的记录

从潮湿接缝退回以后，我没有再沿原路线向前。

博士重新测量了被踩动的石块。旧地图里的位置仍被保留，却标明“表面已经移动，原落脚结论失效”；新位置先保持未知，直到新的高度、法向和不确定范围进入记录。

我从干燥一侧绕开它。每次只执行新预测的第一项，每一步都重新确认接触、地形、全身候选和停止方式。我最终走出了受损通道。

出口外是一条很长的平路。路面没有刚才那些碎石与台阶，却留着两道被许多次通行压出的旧车辙。

我回头看：门槛还在，倾斜的旧修补石没有被我真正踩过，潮湿接缝曾让我滑移，松石已经不在最初的位置。我还主动退回过一次，也绕开了几块从未测清的区域。

“既然我最后到了这里，”我问博士，“能不能把整条通道都写成安全？”

博士把终点与过程分开。到达出口是一项结果；它不能反过来改变旧地图的年龄，不能把 35∘35^\circ35∘ 的坡面变成 25∘25^\circ25∘ 以内，不能让超时求解重新按时完成，也不能证明从未尝试的位置可以通行。

“如果只留下最后一句成功，”他说，“你下一次不会知道自己靠什么通过，也不会知道哪些地方仍然不该再踩。”

第三幕结束时，我仍是同一具四足身体

这次通道穿越从第二幕的静止四足站立开始，到四足低速穿过受损地面结束。身体没有在章节之间重置，也没有在这一章突然增加新部件。

当前结论只属于：

  • 这具四足身体和当前参数版本；
  • 当前近场地图与记录时间窗；
  • 这次穿越采用的低速动作范围；
  • 已经声明的接触、摩擦、驱动和停止条件。

我还没有轮式部件，没有机械臂，没有后来操作任务需要的视觉与力觉，也没有人形身体。将来身体改变以后，同一条路的可通行性必须重新判断；“这一具身体曾在这次穿越中通过”不是道路永久拥有的属性。

一次行动要从哪里开始回看

博士没有从最后一步倒着猜原因，而是按信息产生和被使用的顺序重建：

身体状态与时间
→ 接触意图与实际接触
→ 地形记录与落脚候选
→ 摆动轨迹与整体受力请求
→ 全身控制候选
→ 有限时域预测
→ 驱动与停止方式
→ 实际观察到的结果
→ 地图和行动记录更新

这条链不是九个互相独立的模块名称。后一项能否使用，取决于前一项是否给出了身份明确、仍在有效期内、没有越过边界的结果。

每一段交接究竟要回答什么

表中“谁负责”不是软件包名称，而是一个问题的责任。比如 MPC 可以提出第一项动作，却不能替驱动端证明关节没有饱和；接触状态机可以确认法向接触，却不能替滑移判断证明足端固定。

五个数字怎样改变整条行动链

通道里留下五个不能省略的固定案例。

地图已经比允许时间更旧

某次落脚查询使用的地图年龄为

0.18 s,(1)0.18\,\mathrm s, \tag{1}0.18s,(1)

而本幕声明的上限是

0.10 s.(2)0.10\,\mathrm s. \tag{2}0.10s.(2)

因为

0.18>0.10,(3)0.18>0.10, \tag{3}0.18>0.10,(3)

最先失效的是地形记录。后面即使仍能生成漂亮摆动轨迹或有限全身候选，也不能说它们描述了现在的地面。动作是保持支撑、重新测量或改选。

坡面超过当前身体声明的界限

旧修补石边缘的一格重新测得

θ=35∘,θmax=25∘.(4)\theta=35^\circ, \qquad \theta_{max}=25^\circ. \tag{4}θ=35∘,θmax​=25∘.(4)

坡度余量为

mθ=25∘−35∘=−10∘<0.(5)m_\theta=25^\circ-35^\circ=-10^\circ<0. \tag{5}mθ​=25∘−35∘=−10∘<0.(5)

这一格被标成当前四足身体在本次条件下不可用。它没有被从地图删除，也不被写成“道路整体危险”；原因、身体版本和测量时刻一起保存。

早先那块斜石上另一个 15∘15^\circ15∘ 的几何候选曾通过三项初筛，却没有得到后续动态与执行证明，因此仍属于“候选但未尝试”，不能与 35∘35^\circ35∘ 的明确失败混为同一类。

求解结果来得太晚

一次全身或预测求解耗时

0.035 s,(6)0.035\,\mathrm s, \tag{6}0.035s,(6)

本轮截止时间是

0.020 s.(7)0.020\,\mathrm s. \tag{7}0.020s.(7)

因为

0.035>0.020,(8)0.035>0.020, \tag{8}0.035>0.020,(8)

它即使最后返回数学上可行的数值，也没有当前动作的执行权。记录保留“求解超时”与随后使用的有限停止动作，不能把超时改写成普通成功。

潮湿接缝发生确认滑移

第七章已经由连续切向相对运动和负摩擦余量确认滑移。这里保留三层事实：

  1. 法向接触曾得到确认；
  2. 固定足端的粘着条件随后不成立；
  3. 我减小切向需求、扩大支撑并主动退回。

它不是“接触未知”，也不是“完全失去接触”。分类不同，恢复动作与以后回放时要查的原因也不同。

移动石块以后绕行成功

松石受力后移动，原地图落点失效。我没有继续踩旧坐标，而是更新地图，从干燥侧选择另一条局部路线，并在这次穿越中绕过了它。

“绕行成功”是新路线在这次穿越中的结果；“旧石块位置失效”仍是原路线的记录。两件事同时为真，后一件不会被前一件覆盖。

地图不再只有“安全”和“不安全”

更新后的受损通道地图至少保留下面几种状态：

地图状态 含义 例子
当前条件下可继续使用 对当前四足身体、地图版本和时间窗，已通过声明检查 已知干燥落点与本次成功绕行段
当前条件下不可用 已看到明确硬边界失败 35∘>25∘35^\circ>25^\circ35∘>25∘ 的坡面格
已移动，旧结论失效 表面位置或形状改变，旧坐标不能复用 被踩动的松石原位置

未观察或未尝试 没有足够记录 绕行之外的碎石暗区

同一个区域可以同时带有“发生过滑移”和“后来绕开”等多个标签。强行压成单一颜色，会丢失原因与时间。

行动记录怎样不让最后一个错误覆盖前一个

博士把每一段有意义的动作称为行动记录，把其中出现问题或主动改变计划的一小段称为失败片段。这里使用的是普通记录方式，不建立后面学习系统才需要的正式数据结构。

每条记录至少保存：

字段 要回答的问题
起止时刻 这件事从何时开始、何时结束或仍未结束
身体与地图身份 哪一具身体、哪一版参数和地图
依赖记录 使用了哪些状态、接触、地形、轨迹、控制与预测结果
观察 实际看到了什么，不先替现象命名原因
判断依据 哪个公式、阈值、余量或时间条件被使用
选择的动作 继续、减速、换点、扩大支撑、退回、停止或求助
实际结果 动作完成、未完成、被打断，还是仍然未知
未决不确定性 哪些来源仍无法区分，哪些区域未观察
证据范围 内容、合成计算、模拟器或硬件中的哪一层

若地图过期以后又发现求解超时，两项都保存：

最早使当前落脚失去依据：地图过期
后来仍发现：求解超过截止时间
当前动作：不发送，保持支撑并重新测量

修好求解速度不会让旧地图重新变新；重新测量也不会自动证明求解能按时完成。保留完整时间线，才能知道下一次究竟还要检查什么。

故障分类不是给失败贴一个最终标签

博士把反复出现的问题整理成故障分类，目的是让相同原因可以被搜索、比较和复查，而不是让一个词代替全部细节。

类别 具体含义 常见动作
正常 所有当前必要条件成立，动作得到与范围一致的结果 低速继续一项并重测
边界 余量为零或接近声明界限，尚未越界 减速、扩大余量、加强记录
提前/延迟接触 实际确认时刻早于或晚于计划窗口 失效旧计划，更新接触与轨迹
无支撑 计划要求承载，却没有确认接触或可用支撑 保持其他支撑、回收摆动脚或停止
滑移 接触存在，但固定足端条件不成立 减小切向需求、换点、退回
模型外作用残差 动量变化有模型未解释部分，来源未必可辨 检查状态、接触、参数与外部环境
优化不可行 当前硬条件没有共同候选 改变请求、接触或停止
求解超时/未收敛 候选未在截止前达到声明条件 使用仍有效的有限回退或停止
输入过期 状态、地图、接触或参数不再属于当前时刻 重新取得记录
降低动作要求 为保持硬边界而减速、缩短步长或扩大支撑 保存降低原因与新范围
主动退回 当前向前路线失效，沿已确认路线后退 更新地图并重新规划
主动停止 在越过硬边界前停止发送新动作 保持停止条件和原因
未观察/未尝试 没有足够证据判定 保持未知，不填成成功或失败

一条行动记录可以同时属于“提前接触、地图变化、主动退回”。分类只是帮助查找同类问题的索引，不是互相排斥的最终结论。

证据层级决定最后一句话能说多远

证据层级 至少需要什么 可以支持什么
纸面推演 事件顺序、公式推导、手算与独立算术检查 给定条件中的因果链和固定算例前后闭合
本地合成计算 可重复程序、环境、输入、完整输出与失败注入 指定程序在指定合成输入上的行为
模拟器 模型版本、场景、控制频率、原始日志和回放 指定模拟条件下的运动结果
硬件 身体版本、场地、保护条件、命令、传感记录与停止结果 指定真实试验范围内的现象

我此刻手里只有第一行所列的纸面材料。它们能把到达、滑移、退回和绕行的前因后果接起来，却不能冒充本地程序、模拟器或真实身体留下的记录。

即使以后获得更高层证据，也不能把适用范围删掉。一次硬件通过只属于那具身体、那块地面、那段时间与那组停止条件；它仍不证明未尝试区域永久安全。

这次穿越最终留下了什么

第三幕结束时留下三类面向下一次行动的内容。

一张更新过的受损通道地图

它保存通过位置、明确不可用位置、移动后失效位置、待重测位置、未尝试位置，以及每项结论属于哪具身体和哪个时间窗。

一份四足穿越行动记录

它按时间连接接触安排、地形、轨迹、整体受力、全身候选、预测、驱动、观察、退回和绕行，不让最终到达覆盖中途原因。

一套可继续扩展的故障分类

它区分接触、滑移、模型残差、优化、时间、地图、降低动作、退回、停止和未知，为以后复查相似问题提供共同语言。

这些内容没有改变身体。第三幕结束时，我仍是四足形态。

长路上的车辙提出了下一个身体问题

走出通道以后，地面反而简单了。没有门槛，没有需要逐块选择的碎石，只有一条向远处延伸的平路。

可四足行走仍要不断抬起身体的一部分，再放下，再重新分配支撑。每一步都能完成，时间和能量却在长距离上反复消耗。

两道旧车辙像另一种历史记录：人们曾让滚动的身体在这里反复通过，路也在长期使用中变得更适合滚动。究竟是运动方式适应了道路，还是道路被过去的运动方式塑造成今天的样子，已经很难只看一方回答。

我还没有轮子。博士也没有在这里替我安装。

我只是站在车辙旁提出了第四幕的问题：

坏路已经能走过去，为什么沿着平路走远仍然这么慢、这么费力？

练习

下面六题都从这次完整通道记录中提出。答案要保留最早原因、后续发现与未决问题，不能只写最后一个状态。

练习 1：地图先过期，后来求解也超时，我应从哪里停止这条链

落脚查询使用的地图年龄为 0.18 s0.18\,\mathrm s0.18s，上限为 0.10 s0.10\,\mathrm s0.10s。系统仍继续计算，后来得到一份耗时 0.035 s0.035\,\mathrm s0.035s 的候选，而截止时间为 0.020 s0.020\,\mathrm s0.020s。

沿“地图—落脚—轨迹—全身候选—预测—驱动”追踪最早失效点。当前动作与记录应怎样写，为什么不能只保留最后的求解超时？

练习 2：同样没有固定支撑，滑移与未知接触为什么不能归为一类

片段 A：法向接触已经确认，∣vt∣=0.04 m/s|v_t|=0.04\,\mathrm{m/s}∣vt​∣=0.04m/s，摩擦余量为 −1 N-1\,\mathrm N−1N。

片段 B：计划落地窗口已到，法向作用只有 [0,6,5] N，从未达到连续三份 fn≥8 Nf_n\ge8\,\mathrm Nfn​≥8N。

分别分类并选择动作。两者为什么需要保留不同原因？

练习 3：求解最终返回了数值，为什么 0.035 s0.035\,\mathrm s0.035s 仍是失败片段

当前截止时间为 0.020 s0.020\,\mathrm s0.020s，求解器在 0.035 s0.035\,\mathrm s0.035s 返回一份 KKT 残差很小的候选。状态、接触和地图在这段时间内已经更新。

这份候选还能否发送？应登记为数学不可行、求解超时、输入过期，还是多个类别？说明有限回退的条件。

练习 4：路线记录写着成功，能算模拟器或硬件通过吗

我手里只有这次穿越的事件记录、公式和固定数字的独立复算；其中没有可重复控制程序、模拟器日志或硬件传感记录。

这份记录能支持哪一级结论？如果以后只运行一个数组计算脚本，又能提升到哪一级，仍不能说什么？

练习 5：石头移动以后，地图上的旧点应该删除、改成失败还是保留未知

一个原落点在受力后移动。新测量发现石块落在另一个位置，干燥侧绕行已经完成；石块背后的区域仍未观察。

分别说明旧点、新位置、绕行段和背后区域怎样更新，为什么不能把整片区域统一涂成“已通过”。

练习 6：我已经到达出口，怎样构造“最终成功仍不能证明全路安全”的反例
