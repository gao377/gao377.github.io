第四幕

我已经到了，为什么修理仍未开始——路线复盘与操作缺口

旧工业道路在一扇锈迹斑驳的院门前结束。门内是那座旧工坊，车床、台钳和铺满划痕的工作台都还在原处。

我让轮轴停转，确认四个轮端不再相对地面移动，又让腿部承担起静止时的载荷。计时停止，路线记录写下“到达”。

可是毛坯仍固定在背上。

我可以调整站位，可以向前滚半米，也可以为了门槛抬起一只轮端；却无法卸下毛坯，无法把它摆正，更无法送到机器需要的位置。旧工坊已经在我面前，修理却没有开始。

“我明明完成了到达，”我问博士，“为什么任务还停在门口？”

博士没有立刻回答。他先让我把这一路到底发生了什么说清楚。若连“怎样到达”都只能用一句成功概括，我们也无法分辨现在缺的是最后一段移动，还是一种身体从未拥有过的动作。

换了身体，旧地图也要换一种问法

第三幕离开受损通道时，地图上已经有通过、不可用、发生过边界事件、候选未尝试和未知等记录。它们没有消失，但其中的“可以通过”都带着当时那具四足身体。

现在我的腿端多了轮子，总质量、外形、最低点和接触方式都变了。同一个地点必须重新回答：

旧记录留下了什么 轮足身体还要补问什么 旧结论怎样处理
四足脚曾踩过一块干石 有效轮半径、轮缘与石边是否相撞；这里应滚还是抬轮 原“通过”只保留为四足历史，轮足结论待重算
四足能在窄处原地换向 四个固定轮是否会在转弯中互相擦滑；机身扫掠区是否碰墙 不能沿用原转向能力
某处脚下坡度和落点曾通过 轮端纯滚动约束是否成立，轮足承载与停止是否仍有余量 按当前接触模式重新筛选
某段没有被尝试 当前轮足身体也没有新证据 继续保持未知
某处曾滑移或表面移动 当前表面、湿度与几何是否已经改变 保留旧事件，再建立新版本

我想起博士曾把“找出身体参数与传感记录的对应关系”称为标定。轮半径标定不只影响编码器怎样换成距离，也会改变轮缘能否越过石边、机身最低点离地多高，以及转弯时身体扫过哪里。一个参数同时进入运动、碰撞和里程推算，不能只在某张速度表里改一次。

博士把“某处能否通过必须同时写明是哪一具身体、哪种接触方式和哪个时间条件”称为身体相关的可通行性。道路不会独自带着“安全”或“危险”的标签，身体也不会独自拥有“适应所有道路”的能力。过去的脚塑造了我的旧地图，旧车辙又推动我改变身体。现在究竟是我适应了城市，还是城市把过去身体的选择留在地面上？答案已经同时写在轮子和道路里。

从四足长路到工坊院门，因果链没有跳章

我按发生顺序重新讲述这次运输：

四足身体背负同一件毛坯，建立长路时间与能量基准
→ 我发现长距离反复抬落身体很慢，也很费力
→ 博士为四条腿加装轮端，旧身体状态与参数记录被替换
→ 直路上用编码器、有效轮半径和身体速度互相核对
→ 石质弯道让四个固定轮出现不能同时满足的侧向约束
→ 轮式里程计在转弯与系统偏差中逐渐偏离路线几何
→ 拱形路面使载荷横向转移，全身作用与功率需要重算
→ 低管线挡住整身，滚动切换为卸载、抬轮、越障和同步触地
→ 湿地使纯滚动关系失效，我降低牵引请求并保留滑移证据
→ 干湿交替路段用迟滞和硬边界在滚动、迈步、等待、停止中选择
→ 我在其他边界成立时迈过最后一段，抵达旧工坊院门

这条链里没有另一位主角。背着毛坯走出通道的四足“我”，就是后来获得轮端、成为轮足身体并到达这里的“我”。身体改变需要替换模型和记录，却没有把经历清零。

每个箭头也不是一句文学上的“然后”。它都要说明前一项提供了什么、后一项用了什么，以及边界失败时实际采取了什么动作。例如：

  • 没有确认轮半径与编码方向，直线距离就没有可信比例；
  • 转弯约束发生冲突以后，不能继续把四轮全都写成理想纯滚动；
  • 里程计不确定性增大以后，下一段路线判断要带着这份范围；
  • 卸载不足时，抬轮动作不得开始；
  • 湿地牵引不足时，较低能耗不能批准继续滚动；
  • 最后选择迈步，只属于当时那组预测、时限和停止条件。

一条记录要让不同速度的事情排成同一段经历

轮轴编码器、身体状态、接触判断、能量累计和模式决定并不在同一时刻产生。如果只把它们按文件行号拼在一起，就可能把减速后的轮速拿去解释减速前的滑移。

这次路线记录至少要对齐：

记录内容 必须附带的身份 它回答什么
身体状态 身体版本、坐标系、估计时刻、不确定范围 当时机身和关节在哪里、怎样运动
轮速与编码器 轮端身份、计数区间、比例与方向版本 每只轮在该区间转了多少、方向如何
轮式里程计 起止状态、积分方法、参数版本、协方差 相对位姿怎样累计，可信范围多大
接触与滑移 轮端、接触模式、迟滞窗口、有效速度范围 当时是承载、滚动、摆动、触地还是滑移
全身与牵引作用 来源模型、承载分配、限幅前后值 请求了什么，实际允许了什么
能量与功率 积分起止时刻、包含的执行器和待机项 哪一段动作花了多少，不遗漏切换
模式与决定 候选集合、硬边界结果、迟滞状态、有效期 为什么滚、迈、等、停或求助
毛坯状态 固定点、质量、是否移动、是否仍在背上 运输对象有没有改变或脱离
结果与停止原因 观察时刻、动作是否完成、最后安全状态 发生了什么，为什么结束或降级

对一段从观察到动作再到结果的链，至少要满足可核对的先后关系：

tobservation≤tdecision≤tcommand<tresult.(1)t_{\mathrm{observation}} \le t_{\mathrm{decision}} \le t_{\mathrm{command}} < t_{\mathrm{result}}. \tag{1}tobservation​≤tdecision​≤tcommand​<tresult​.(1)

如果设备使用不同的时钟，还要保存时钟对应关系和误差范围。式 (1) 不是说所有计算都在同一个采样点完成，而是要求我们能确认：决定没有使用未来数据，结果也确实发生在动作之后。

某一项没有时间戳时，不应把它猜到最方便支持当前结论的位置。记录可以保留这个数值，却必须把对应因果关系标为未知。博士把沿统一时间语义重建“观察—决定—动作—结果”的工作称为因果回放。

到达不是一块盖住旧记录的白布

最终状态可以写成：

地点：旧工坊院门
身体：同一具轮足身体，当前参数版本已声明
毛坯：仍固定在背部安装点，未卸下、未摆正
运动：已停止，当前接触与静止条件在路线记录中成立
路线结果：纸面路线在给定条件下到达
证据范围：事件记录、公式与独立手算

但这条“到达”不能覆盖以下事实：

  • 初装轮端时出现过半径、比例或方向的标定问题；
  • 固定轮在弯道上出现过约束冲突与擦滑；
  • 轮式里程计保留随机扩散和系统漂移；
  • 拱形路面发生过载荷转移，某些请求需要重新分配；
  • 管线处切换过滚动与迈步，失败分支没有因此消失；
  • 湿地出现过滑移，牵引请求曾被降低；
  • 最后一段放弃了能量更低但牵引余量为负的滚动候选；
  • 绕行之外、没有重新观察的位置仍然未知。

一次成功只能新增“在这组条件下，这条实际采用的路径在路线记录中到达”。它不能把未尝试变成成功，也不能把早期错误改写成从未发生。这样，下一次半径再次异常时，我们能找到相似记录。下一次在湿地偏航时，也不会把所有误差都归咎于里程计。

同样是“位置不对”，原因可能在不同层

为了让以后能查找相似问题，博士在第三幕故障分类的基础上补充轮足身体特有的类别：

类别 这一次具体指什么 不能与什么混淆 当前动作
参数或编码错误 有效轮半径、计数比例、安装方向或符号不符 真实地面滑移 停止沿用旧换算，重新检查标定与映射
滚动约束失效 接触仍在，但轮地相对运动不再满足声明的纯滚动关系 编码器比例错误 降低请求，检查滑移与接触模式
固定轮转弯擦滑 多个固定轮在同一转弯中产生不可同时消除的侧向残差 单只轮的偶发噪声 降低转弯请求或改变接触方式
里程计漂移 相对位姿误差随积分累计，随机或系统成分逐渐显现 某一时刻已经确定的唯一根因 扩大不确定范围，寻找独立约束并回放来源

降级事件 为守住硬边界而主动减速、改迈步、退回、等待或停止 无原因的性能下降 保存触发条件和恢复条件

这些类别可以同时出现，也可以存在因果关系。有效轮半径偏大可能造成长期里程系统偏差；湿地滑移也可能造成里程突变；“里程计漂移”描述累计结果，不自动证明根因是哪一个。

分类还应保留最初观察。例如“轮速上升而身体速度没有同比上升”先是一项现象；只有核对有效轮半径、编码方向、低速有效性和接触以后，才有资格进一步写成滑移或参数错误。过早命名会让后续检查只寻找自己已经相信的原因。

这份旅程记录还缺什么

我摊开手里的路线记录、公式、给定数字和独立手算。它们足以检查旅程的前因后果是否接得上、单位是否相容、每个分支是否守住声明的硬边界。

它没有保存一套可重复运行的闭环控制程序，没有模拟器场景与原始日志，也没有真实轮足身体的传感器和命令记录。因此不能声称：

  • 轮式里程计已经实时运行；
  • 全身控制或风险感知预测已经闭环通过；
  • 这条路已由模拟器或硬件验证；
  • 这次到达已经构成工坊安全授权。

这也不是后续学习系统所需的正式训练样本。正式样本还要定义观察、动作、时间边界、任务条件、终止原因、版本和可重复读取方式；我只把它保留为普通运输记录，不为一个时髦名称假装已经具备数据管线。

门内缺少的不是又一米路

我把路线记录最后一行重新读了一遍：位置已经是旧工坊院门，毛坯仍在背上。即使我把里程误差再缩小一些，再向前滚一米，毛坯仍不会自己离开固定点；即使我能在门槛前更稳地迈步，也不能把它摆到台钳中央。

我终于能把问题说得更具体：

我怎样让身体的一部分接近毛坯，承受它的重量，解除背部固定，把它带到工作台上，再以需要的位置和方向放下？

“从环境中取得一个物体，改变它的位置、方向或接触关系，使它进入任务需要的状态，”博士说，“这类问题叫作操作。”

这个名字没有让新的身体部件凭空出现。此刻我仍只有轮足身体，仍没有能伸向毛坯的手段，也没有抓持、观察物体和感受接触作用所需的能力。第五幕将从我的这个问题开始：博士会在任务需要的范围内引导我增加新的身体部件，而每次增加都必须重新回答运动链、负载、碰撞、感知和停止怎样改变。

轮子让我沿一座由过去通行塑造的城市走得更远；工坊门口却提醒我，进入世界不只意味着在它的表面移动，还意味着能够与其中的事物建立新的关系。

练习

下面六题发生在我停在旧工坊院门、准备封存本次运输记录的时候。每题都要保留同一件毛坯、同一具轮足身体和同一证据范围。

练习 1：四足时走过的干石，为什么现在不能直接标成通过

第三幕地图里，一块 15° 的干燥斜石曾通过四足脚的几何初筛，附近绕行段也在那次穿越里走过。现在轮足身体的地图查询仍返回“可通过”，却没有轮半径版本、轮缘碰撞、固定轮转弯扫掠区或滚动/迈步模式字段。

判断这条“可通过”能否交给当前路线决定。说明旧记录应怎样保留，我下一步至少要重新检查什么。

练习 2：没有时间的轮速还能解释滑移吗

路线记录中依次出现：12.00 s 的身体速度、一个没有时间戳的轮速数组、12.08 s 的滑移标志和 12.10 s 的牵引降低命令。四项都属于当前轮足身体，但轮速所用设备时钟与主时钟的对应关系也没有保存。
