第六幕

抓住一次以后，我凭什么相信它没有滑——抓取执行、验证与重新抓取

我先测试上一章保留下来的候选。第一次闭合后，夹爪开口和电流都很稳定；吊架缓慢转移 30 N30\ \mathrm N30 N 时，相机却看见毛坯相对指尖向下移动。第二次，夹爪一直合到机械最小开口，电流也在末端升高，但腕部和吊架都没有感到物体作用。

我让吊架恢复原来的承托：“开口停住、电流上升，为什么一次是滑动，另一次甚至可能什么也没抓到？”

博士说，同一个信号可以由不同原因造成。开口停止可能是碰到物体、夹具或机械限位；电流上升可能来自夹紧、卡滞或末端硬停止。只有把动作阶段与多种观测对齐，才能知道它们在解释哪一件事。

他把预抓取、接近、闭合、受支撑载荷转移、保持和退回组织成的离散流程称为抓取执行状态机；把多信号共同支持“物体仍按预期留在手中”的检查称为持物验证。

候选只有经过动作，才会遇到真实失败方式

解析筛选假设了接触位置、法向、摩擦、物体位姿和夹爪作用。执行会暴露这些假设的偏差：

  • 预抓取位姿不准，闭合时只有一侧先碰到；
  • 夹指提前碰到夹具或物体边缘；
  • 物体被接近气流、轻触或单边接触推走；
  • 夹爪合到机械限位，实际没有接触物体；
  • 接触建立但摩擦不足，载荷转移时滑动；
  • 相机被夹爪遮挡，无法判断相对运动；
  • 腕部、夹爪与吊架的时间或符号不一致。

因此“抓取成功”不能是 close 命令返回一个完成标志。每个阶段都要有进入条件、连续监测、通过条件、退出条件与恢复动作。

六个阶段各自承担什么

预抓取

我到达物体外的预抓取位，夹爪保持张开。进入前必须确认物体位姿、场景和候选仍有效；通过条件是末端误差、碰撞余量和可见性都在声明范围内。

接近

我沿已检查走廊低速靠近。视觉或位姿状态过期、特征丢失、腕部出现意外作用、碰撞余量不足时立即停止。接近完成只说明到了闭合起点，不说明已经接触。

闭合

夹指以受限速度合拢，同时记录开口、请求/实际作用、电流、腕部力旋量和物体相对位姿。若在预计开口之前一侧作用上升，可能是提前碰撞；若到机械最小开口仍无物体证据，可能是空抓。

受支撑载荷转移

吊架只把声明的 30 N30\ \mathrm N30 N 缓慢转移到夹爪，毛坯仍不脱离支撑。此阶段检查接触作用、夹爪余量、物体相对运动和吊架变化是否在同一时间窗内一致。

保持

在有限时间内保持相同受支撑条件，观察位移或作用是否持续漂移。瞬间未滑不能替代连续样本；但保持时间再长，也不授权超过当前载荷和支撑范围。

退回

无论通过还是失败，都先让吊架恢复承托，再撤去夹爪的载荷责任。确认作用下降和物体稳定后，才张开并退回。若物体位置已变，旧候选和旧接近路线随之失效。

五类信号互相补足，也可能互相冲突

夹爪开口

开口停止在与物体厚度和指垫压缩相符的位置，支持“可能接触”；停止在机械最小值，支持空抓或极薄物体；停止得过早，则可能碰到边缘或夹具。它不能单独说明摩擦是否足够。

电流或估计夹紧作用

上升支持夹爪受到阻碍，但还要排除机械限位、传动摩擦和卡滞。由电流换算作用依赖温度、传动与标定，不能把原始安培数直接当成指尖牛顿。

腕部力旋量

它能看到整只工具与外界的合成作用，却不直接告诉我两侧各自接触多少。偏置、工具重力和夹具碰撞都可能进入记录。

视觉相对运动

应观察物体相对夹爪或指尖的位移，而不是只看物体在世界中是否移动。底盘锁定与相机关系仍可能有微动；遮挡或位姿跳变时，视觉应标为未知而不是自动判滑。

吊架载荷

它说明有多少重量从支撑系统转移出去。吊架减少约 30 N30\ \mathrm N30 N 与夹爪/腕部相应变化应在时间上对应；若只有吊架变化，可能是吊架自身动作或同步错误。

这些信号不必每次给出完全相同的数值，但必须在物理方向、量级和时序上能够共同解释。冲突时输出“暂时无法判断”，不能投票取多数。

怎样定义滑移候选

令物体相对夹爪的平移为

pog(t),p_{og}(t),pog​(t),

相对位移为

Δpog=pog(t)−pog(t0).\Delta p_{og}=p_{og}(t)-p_{og}(t_0).Δpog​=pog​(t)−pog​(t0​).

滑移判断应区分：沿重力方向的持续移动、绕接触的转动、视觉噪声和整具手臂的共同运动。本章不把一帧跳变当成滑移；要求多个有效帧呈同向变化，并由吊架转移或接触作用变化支持。

可使用进入与退出不同门槛的迟滞。例如连续三帧相对速度超过给定门槛形成滑移候选，连续若干帧低于更小门槛才记为停止滑移。具体门槛必须来自当前相机噪声、帧率和物体余量，不能从本章复制到真实设备。

若视觉被完全遮挡，不是“没有看见滑移”，而是“滑移状态未知”。在原毛坯不能自由举升的前提下，未知意味着恢复支撑，不意味着继续转移。

失败码要连接到不同恢复动作

状态 PREGRASP：
  位姿/路线有效且无碰撞 -> APPROACH
  否则 -> REPERCEIVE 或 STOP

状态 APPROACH：
  到达闭合起点 -> CLOSE
  意外作用/遮挡/过期 -> RETREAT

状态 CLOSE：
  开口、作用和物体证据一致 -> LOAD_TRANSFER
  到机械限位且无物体证据 -> EMPTY_GRASP
  过早作用或夹具接触 -> PREMATURE_CONTACT
  过载 -> RESTORE_SUPPORT

状态 LOAD_TRANSFER：
  只转移 30 N，并持续同步检查多信号
  滑移/冲突/过载/感知丢失 -> RESTORE_SUPPORT
  条件连续通过 -> HOLD

状态 HOLD：
  相对运动和作用保持在声明范围 -> RESTORE_SUPPORT 后记为受支撑测试通过
  漂移或状态未知 -> RESTORE_SUPPORT

状态 RESTORE_SUPPORT：
  吊架恢复承托、作用下降、物体稳定 -> RETREAT
  无法恢复 -> STOP 并请求人工处理

状态 RETREAT：
  张开、退回、重新感知；旧候选按场景变化决定是否失效

重新抓取不是立刻重复相同命令。它先根据失败码决定：空抓要重新感知物体位置；提前碰撞要检查场景和接近路线；滑移要换接触、作用分配或继续支撑；传感冲突要修复时间与坐标后再试。

每次结果都要保存身体、工具、物体、摩擦下界、场景、候选、控制与时间版本。否则一次通过无法说明后来换过指垫或物体姿态后是否仍适用。

原毛坯的授权没有改变

完成受支撑测试，只证明在这一次 30 N30\ \mathrm N30 N 载荷切片和有效观测下，流程没有发现滑移、空抓或过载。上一章算出的保守全重要求仍超过夹爪上限，所以原毛坯继续由吊架或台面承托。

我现在能够可靠地把毛坯保持在支撑中，并在失败时恢复。接下来，夹爪要配合固定工艺设备，把毛坯送进加工夹具；最后几毫米不再只是“抓紧”，而是接触模式不断变化的插入问题。

练习

四题都只允许 30 N30\ \mathrm N30 N 受控载荷转移。

练习 1：开口和电流稳定，物体却在向下走

夹爪开口与估计作用保持稳定；吊架载荷下降 30 N30\ \mathrm N30 N。连续有效视觉帧显示物体相对夹爪在 0.18 s0.18\ \mathrm s0.18 s 内向下移动 3.6 mm3.6\ \mathrm{mm}3.6 mm。判断下一状态和失败类型。

练习 2：电流在最小开口处上升

夹爪合到机械最小开口后电流上升；腕部没有出现与物体接触相符的变化，吊架载荷不变，视觉显示毛坯仍在原位。判断更像有效抓取、空抓还是滑移。

练习 3：视觉被完全遮住，其他信号暂时正常

载荷转移到 20 N20\ \mathrm N20 N 时，夹爪开口、作用和吊架记录一致，但物体相对位姿因遮挡变为未知。判断能否继续到 30 N30\ \mathrm N30 N 并记为通过。

练习 4：吊架与腕部的时间线对不上

吊架记录在 t=0.20 st=0.20\ \mathrm st=0.20 s 已减少 30 N30\ \mathrm N30 N，腕部和夹爪作用直到 t=0.29 st=0.29\ \mathrm st=0.29 s 才变化；系统声明的残余同步误差不应超过 8 ms8\ \mathrm{ms}8 ms。判断应解释为抓取成功、传感冲突还是物体惯性，并决定下一步。
