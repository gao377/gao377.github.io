第十一幕

图像、作用和接管，怎样属于同一件事——多模态对齐、接触事件与成功谓词

我给每条 applied action 找最近的图像、触觉和作用记录，很快拼出一张“完整”表。可回看其中一段时，我发现图像拍在人工接管之后，动作却施加在接管之前；接触刚刚 reset，插值又把 reset 两侧的作用连成一条平滑曲线。

表格没有空格，经历却已经被我拼错了。

博士把这种“不同传感与动作记录怎样属于同一个事件”的问题称为多模态 alignment；当接触、人工接管和任务结果也必须保留时，它同时需要 contact-rich data、intervention Event 与 success predicate。

“最近”先要回答是哪一种时间

一个观测可能经历曝光、采样、接收和处理；一个动作则经历计算与施加：

texposure→tsample→treceive→tcompute→tapply.(1)t^{exposure}\rightarrow t^{sample}\rightarrow t^{receive}\rightarrow t^{compute}\rightarrow t^{apply}. \tag{1}texposure→tsample→treceive→tcompute→tapply.(1)

图像表达的是曝光区间内的光，触觉表达的是采样时刻附近的接触，动作影响世界则从 apply 开始。若先用 receive time 排序，网络晚到的旧图像可能被配给较新的作用。不同 clock domain 还要先通过带不确定度的映射：

t^B=fA→B(tA),∣tB−t^B∣≤ϵclock.(2)\hat t_B=f_{A\rightarrow B}(t_A),\qquad |t_B-\hat t_B|\le \epsilon_{clock}. \tag{2}t^B​=fA→B​(tA​),∣tB​−t^B​∣≤ϵclock​.(2)

只有时间差、曝光/积分区间和 ϵclock\epsilon_{clock}ϵclock​ 都满足当前用途，才有资格继续对齐。本幕不设一个通用“最近多少毫秒就算同一条”的阈值；视觉、触觉、作用和慢速任务结果的可接受窗口本来就不同。

离散事件是不能插过去的墙

接触开始、确认、滑移、释放，人工接管的 request/accept/apply/release，身体或标定 revision 切换，都是 event barrier。它们两侧的连续状态可能使用不同动力学、owner 或 action 语义。

因此对齐先划事件窗，再在窗内做 hold、插值或短时外推：

换算时钟并绑定所有相关版本
按事件发生时刻排序，而不是按到达顺序排序
用任务、接触、接管和版本事件划出不能跨越的窗口

对每个训练或复核样本：
    只在同一个事件窗口内选择不同模态
    保存曝光或积分区间以及时钟不确定度
    超过允许差异时，将对应模态标为 MISSING 或 LATE
    样本之间若存在重置或事件屏障，不做插值
    保留乱序到达事实和原始时间戳

低矮防水巡检身体的接触点、轮足历史的轮地状态和当前人形的手指/足底状态，各自绑定 body/contact schema。它们可以共享“接触开始”这一抽象 Event，却不能强行共享一个触点数组。

轨迹相似不是任务成功

一次安装、通过或接管的结果要由任务专属 success predicate 判断。抽象地：

S=Sidentity∧Saction∧Scontact∧Soutcome∧Sevidence.(3)S= S_{identity}\land S_{action}\land S_{contact}\land S_{outcome}\land S_{evidence}. \tag{3}S=Sidentity​∧Saction​∧Scontact​∧Soutcome​∧Sevidence​.(3)

例如末端轨迹接近示范，只能说明几何路径相似。若接触时发生滑移、人工已接管、零件没有锁止，或者 outcome 未观测，成功仍分别是 fail、partial 或 unknown。成功谓词必须保存版本、分子/分母口径和所需观察；不能在训练后为了提高分数临时改定义。

干预也不只是一位人员出现在画面里。要区分 request、accept、apply 和 release，并保存 actor 与 authority；否则接管后的安全动作可能被错误标成自主策略的优秀示范。

练习

练习 1：图像最近，却跨过了接管

applied action 在 t=10.000 s，接管 apply 在 10.010 s，最近图像的曝光区间是 10.012—10.028 s。说明图像能否作为该自主 action 的视觉输入，以及怎样记录缺口。

练习 2：作用晚到，但 sample time 合适

一条腕部作用记录在网络中晚到 80 ms，其 sample time 与接触事件同窗，clock mapping 不确定度也满足当前回放合同。判断能否按 receive order 丢弃，及对实时训练标签还需检查什么。

练习 3：轨迹重合，零件没有锁止

末端轨迹与示范高度相似，接触中没有超限，但锁止传感器缺失、人工在结尾扶住零件。给出 success/partial/fail/unknown 的合理记录和理由。
