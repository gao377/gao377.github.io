第五幕

关节依次转动，手会走到哪里——机械臂正运动学

初始坐标关系登记以后，我来到开阔测量台。博士没有给我一个“伸到毛坯”的目标，只让我把七个关节停在一组远离限位的角度，再预测夹爪工具中心点会出现在哪里。

每个关节单独低速检查时，正方向都与登记一致；把七个角度一起送进模型后，屏幕里的夹爪却偏到测量标记外。有人建议直接把七段连杆的位移向量相加，再调整目标点抵消偏差。

我想起第一幕里，腿的第二段并不是相对机身独立摆放，而是接在第一段已经转过的坐标系后面。现在的七段手臂也一样：越靠后的连杆，会继承前面所有关节已经造成的旋转和平移。

我问：“如果每一段都对，为什么把它们相加还会错？”

博士这才把“已知关节配置，求各连杆和末端位姿”的问题称为正运动学。它不替我选择关节怎样转，只忠实回答：按当前模型与顺序，这组关节角会把手带到哪里。

正运动学的输入和输出先要固定

完整操作臂的关节配置为

q=[q1q2⋯q7]T∈R7,q= \begin{bmatrix} q_1&q_2&\cdots&q_7 \end{bmatrix}^{\mathsf T} \in\mathbb R^7,q=[q1​​q2​​⋯​q7​​]T∈R7,

七个转动关节都用弧度。正运动学输出锁定底盘坐标系 {B}\{B\}{B} 中的 TCP 坐标系 {E}\{E\}{E}：

BTE(q)∈SE(3).{}^{B}T_E(q)\in SE(3).BTE​(q)∈SE(3).

它同时包含位置和朝向，不是只有一个三维点。函数还应输出每个中间连杆位姿

BTLi(q),{}^{B}T_{L_i}(q),BTLi​​(q),

因为以后自碰、环境碰撞和故障定位都需要知道整条手臂在哪里。

计算前必须确认：

  • 数字身体模型与当前身体配置一致；
  • 机械臂标定登记中的底盘—安装座关系有效；
  • qqq 的名称、顺序、单位、时刻和零位一致；
  • 每个 qiq_iqi​ 有限且位于允许检查范围；
  • TCP 定义与夹爪当前安装一致。

一个 4×44\times44×4 矩阵数值合法，不代表这些前提已经成立。

把同一件事写成逐节变换

平面式容易看见累计角，七关节三维手臂则更适合逐节组合刚体变换：

BTE(q)=BTMMTL1(q1)L1TL2(q2)⋯L7TE.{}^{B}T_E(q) = {}^{B}T_M {}^{M}T_{L_1}(q_1) {}^{L_1}T_{L_2}(q_2) \cdots {}^{L_7}T_E.BTE​(q)=BTM​MTL1​​(q1​)L1​TL2​​(q2​)⋯L7​TE​.

每个矩阵只回答相邻两级坐标系的关系。右侧 TCP 固定变换说明最后一个关节坐标系与实际夹爪工具中心不一定重合。

中间结果不能只在调试时临时打印；它们是可检查产物。若最终 TCP 偏差出现，可以先找到从哪一节开始与独立测量不一致，而不是把七个关节一起调。

相邻关系怎样被压缩成四个参数——DH 表

对连续转动或移动的低副机构，Denavit 与 Hartenberg 提出过一种用矩阵统一记录相邻连杆关系的记法。今天常说的 DH 参数延续了这个思想，但教材和软件还存在“标准”“改进”等不同约定，不能只写四列数字而不声明乘法顺序。

本章若采用标准 DH 顺序，单节变换定义为

DH 表的优势是紧凑，也容易与传统机械图对照；风险是坐标系放置规则和变体约定必须完整保存。若一份表不声明标准还是改进 DH、角度单位和零位来源，它就不是可复现模型。

原始论文记录为 J. Denavit 与 R. S. Hartenberg 1955 年的 A Kinematic Notation for Lower-Pair Mechanisms Based on Matrices。这条历史只支持“矩阵化记法的来源”，不意味着今天所有 DH 变体都与原文写法相同，也不证明当前七关节参数正确。

指数积怎样直接保存关节轴

另一种表达不必为每一节选择 DH 坐标系，而是在零姿态中记录每个关节轴。对第 iii 个转动关节，在空间坐标系表达的单位轴为 ωi\omega_iωi​，轴上取一点 rir_iri​，则对应运动旋量为

工作空间不是把七个关节范围相乘

所有允许关节配置的集合称为配置空间。把这些配置通过正运动学映射到 TCP 能达到的位置和朝向，才形成工作空间。

关节限位、连杆长度和轴向会限制几何可达范围；不同姿态可能到达同一 TCP；某些位置可达但朝向不可达。以后还要叠加自碰、环境碰撞和奇异性，所以“几何工作空间内”仍不等于可执行。

正运动学只做单向映射：

q⟼BTE(q).q\longmapsto{}^{B}T_E(q).q⟼BTE​(q).

它通常容易计算，却不能反过来保证给定目标一定存在关节解。等目标真正出现时，我还要反过来寻找关节姿态，逆运动学也会因此进入我的判断。

运动学模型怎样计算并自查

输入：
  数字身体模型
  机械臂标定登记
  q[1..7] 与共同 sample_time
  选定且声明约定的 DH 或 POE 参数

1. 核对身体、标定、关节顺序、TCP 与参数版本。
2. 若 q 维数错误、含非有限值、单位未知或超出允许检查范围，停止计算并说明原因。
3. 从 {}^B T_M 开始。
4. 按冻结关节顺序逐节右乘相邻变换，并保存每个连杆位姿。
5. 组合固定 TCP 变换，输出 {}^B T_E(q)。
6. 检查每个旋转块的正交性、行列式与齐次矩阵末行。
7. 对零姿态、单关节姿态和三关节切片做独立几何复算。
8. 在代表性姿态比较 DH 与 POE；差异超界则模型不可用。

失败：
  返回无效状态与首个明确原因；
  不用单位阵、零角或旧配置静默补值。

输出：
  机械臂运动学模型
  TCP 与七段连杆位姿、使用版本、检查残差和有效用途

每个关节只需一次固定大小的刚体变换和矩阵乘法，因此对 nnn 关节的前向递推为 O(n)O(n)O(n)。数值成本不高，真正危险的是以错误顺序快速算出一个看似正规的矩阵。

关节角终于能回答“手在哪里”，却还不会回答“手怎样动过去”

正运动学把每个关节的局部动作接成了一个整体结果。它也让我看到，身体并不是七个部件的简单集合：前面一处很小的转动，会改变后面所有连杆解释“前方”和“上方”的方式。

这种整体性不是神秘关系，而是可以沿每个坐标系逐步复算的组成。数字模型因此获得了一个新的用途：在声明配置内，从当前关节角计算整臂位姿。它仍没有证明姿态无碰、关节可承受、标定真实或路径可执行。

下一步的问题来自局部变化：手只想移动一点，七个关节的微小变化怎样共同造成这段运动？

练习

下面四题使用同一条运动链。判断应当优先修模型、修变换、停止处理错误输入，还是继续复算，不能靠修改任务目标掩盖身体关系错误。

练习 1：三段连杆为什么不能分别只转自己的角度

三关节切片取 [0.28,0.25,0.16] m，角度为 [30°,-45°,20°]。用累计角计算 TCP 的 (x,y,ϕ)(x,y,\phi)(x,y,ϕ)；再比较错误方法“每段只使用自己的关节角”得到的约 (0.569615,0.017947) m。

决定偏差出现后应修改目标还是修正运动链组成，再说明你会用哪一条不同来源的证据核对判断。

练习 2：交换第二、第三关节，末端朝向为什么还能一样

同一切片中，正确顺序是 [30°,-45°,20°]；错误记录把第二、第三关节交换成 [30°,20°,-45°]。两者角度和都为 5°。

错误顺序给出的 TCP 约为 (0.562575,0.345456) m。比较正确位置，解释为什么只核对末端朝向会漏掉关节顺序错误，并决定下一步检查。

练习 3：安装座变换方向错了，我能否移动目标补偿

七关节链在安装座坐标系中通过零位和单关节复核；组合到底盘坐标系时，程序把 BTM{}^{B}T_MBTM​ 用成 MTB{}^{M}T_BMTB​。所有目标因此出现结构化偏差。

判断应修改每个目标、修改 TCP，还是修正安装边方向。给出至少两个不依赖新目标的复核。

练习 4：DH 与 POE 在零姿态相同，是否已经足够

两种模型共享同一个零姿态 MMM，所以 q=0q=0q=0 时完全一致；当只转第四关节 0.20 rad 时，末端位置开始分离，而旋转块仍各自正交。检查发现 DH 表与 POE 关节轴对第四关节使用了不同正方向。

判断哪一个模型可以直接采用。说明零姿态、单关节和多关节测试分别能发现什么，以及为什么“矩阵合法”不能决定谁对应真实身体。
