第六幕

一步做对了，为什么整件零件仍会失败——任务编排、恢复与成功谓词

维修座已经被我从夹具里提起。侧面的抓法很牢，短距离搬运也没有滑动；可我一转向装配台，手腕便会撞上护栏，衬套的轴线也对不准插入方向。空中绕路没有用：维修座和手腕的相对姿态没有改变。

我问博士：“这只手明明抓住了，路也没有被堵死，为什么还是到不了最后一步？”

他没有先让我换一条轨迹，而是让我把整件事重新说一遍：在哪里抓、何时有支撑、何时可以松手、从哪一侧再靠近、怎样确认已经插好。原来，一次抓取的成功不等于整个任务已经有解；每一步分别做对，也不等于它们可以无条件接在一起。

有限状态机把“我现在在哪里”写清楚

我先把过程写成一串状态：

已抓稳 -> 已抬起 -> 已放稳 -> 已松手 -> 已换边抓取 -> 已插入 -> 已复查

博士把这种“状态加转换条件”的组织方式称为有限状态机（FSM）。每个箭头都必须回答：维修座身份是否一致，支撑是否仍可靠，场景、抓取、计划和感知版本是否仍有效。失败也不能一律跳回开头：

插入卡滞 -> 恢复支撑 -> 重看孔位 -> 重新规划
感知丢失 -> 保持不动 -> 重新观察
支撑状态未知 -> 停止并求助

FSM 容易记录，却会随着任务和恢复分支增多而迅速膨胀。若每条边各自藏着一套恢复规则，还会漏掉本应作废的旧状态。

行为树把顺序、选择和恢复组成层次

博士又把检查和动作摆成一棵反复从根部查看的行为树（BT）。动作可以返回“运行中、成功或失败”；顺序节点要求前一项成功才向后走，选择节点在失败后转向备选或恢复，条件节点则每次重新检查世界，而不是永远相信旧结论。

顺序
  当前孔位仍有效？
  当前抓取与维修座身份一致？
  插入动作
  深度、姿态、作用和量规都通过？

树形结构只让责任更容易组织，并不会自动保证安全。条件节点若读了过期孔位，恢复分支若绕过作用上限，行为树一样会把错误安排得井井有条。Colledanchise 与 Ögren 的行为树导论系统说明了这种执行结构。

一种抓法能搬运，却不能把整件事做完

现在的问题不在“插入”这个动作本身，而在动作之间怎样交接。当前抓法通过了闭合、抬升和短程保持检查，却在插入前的连续检查中失败：若保持物体相对手腕的姿态，手腕会碰护栏；若硬把衬套转向孔轴，又会越过关节可达范围。

我说：“那我先把维修座放下，松手，从另一侧再抓呢？”

博士这才把离散任务选择与连续几何、运动、动力学和接触检查互相调用的方法称为任务与运动规划（TAMP）。连续检查失败不是故事的终点，它要把“哪一种绑定、因为什么失败”送回离散层：这次应换抓法，不是把同一抓法的空中路线再弯一次。

把抓取和支撑姿态连成一张再抓取图

我们把每一种有实际含义的处境画成节点：

  • 在夹具上受支撑、保持当前抓法；
  • 已抬起、保持当前抓法；
  • 在中间支撑面稳定放置；
  • 已松手，并且另一侧仍可重新靠近；
  • 已采用与插入方向相容的新抓法；
  • 已插入，等待成功谓词复查。

节点之间的边不是一句“移动”。抬起边要调用抓取与持物检查；搬运、靠近和退回边要调用自由空间规划；推、放、插入边还要调用上一章形成的 dynamic-contact-motion-plan-v0；松手边必须先确认物体在声明过的支撑面上稳定，并且放置没有堵住下一次视线和靠近方向。每条边在交给身体前后还要经过版本、动作生命周期和实际结果的执行检查。

博士把这种由抓法、支撑状态和可行转换组成的结构称为再抓取图。早期再抓取研究已经把“一次抓不完，就借助稳定放置换抓”写成图搜索问题，可参见 Tournassoud、Lozano-Pérez 与 Mazer 的经典工作。图上的路线由动作次序组成，每条边仍需当前身体和世界的连续证据。

连续检查失败以后，离散搜索怎样继续

搜索从当前状态出发，把待检查节点放进 frontier。每次给“放置、再抓取、插入”绑定具体支撑位、抓法、位姿和接触模式，再调用相应检查：

frontier ← 初始离散状态
while frontier 非空且预算未耗尽:
  node ← 取出一个待检查状态
  for action in 当前可用动作:
    candidate ← 绑定抓取、放置、位姿和模式参数
    result ← 调用对应连续检查
    记录 candidate、依赖版本、result 和失败类别
    if result 可行:
      若成功谓词成立，返回计划与搜索记录
      否则把后继加入 frontier
    else:
      只排除当前绑定；根据失败类别决定是否排除动作模式
若场景、身体、物体、抓取或模型版本变化，作废受影响缓存
返回无可行方案、超时或信息不足

“当前抓法加这个支撑位会碰撞”只排除这一组绑定；“所有允许的支撑位都无法稳定放置”才可能排除当前放置模式。冻结版本、枚举范围和预算内的绑定都已耗尽，可以返回“当前条件下无可行方案”；支撑面是否被占用都看不清时，只能返回“信息不足”。前者要求换目标、身体能力或候选范围，后者要求先重新观察，不能混成一句“规划失败”。

这次搜索留下 tamp-search-record-v0：frontier 取出过哪些节点，连续检查为何通过或失败，何时停止，以及每个结论依赖哪些版本。于是“放稳—松手—换边靠近—再抓—重做自由空间与动力学/接触规划—插入”不再是一串凭感觉补上的动作。

缓存什么时候还能用，什么时候必须作废

连续检查很贵，已经验证过的抓取、放置、逆解和运动边可以放进缓存。但缓存键不能只有“起点到终点”，还要包含身体、负载、物体、抓法、支撑、场景、目标和检查器版本。

若只是离散搜索换了顺序，而这些依赖没有改变，旧结果可以复用。若手腕外形、碰撞余量、维修座位姿或抓法变了，相关正、负结果都应失效；否则旧的“可行”会撞上新障碍，旧的“不可行”也会挡住如今已经出现的通路。作废应沿依赖关系传播，而不是粗暴地忘掉一切，也不能为了省时间继续相信受影响的边。

PDDLStream 与 LGP/KOMO 改变了怎样的接口

同一责任可以有不同组织方法。PDDLStream 把“已抓取、已放稳”这类符号事实交给任务搜索，把位姿、抓法、逆运动学（IK）和运动候选交给黑盒流按需生成；流的结果成为新事实，连续失败则带着当前绑定返回。它需要清楚的谓词、动作、流输入输出与版本，不要求离散层自己懂所有几何细节。可参见 Garrett、Lozano-Pérez 与 Kaelbling 的PDDLStream 论文。

LGP/KOMO 则先探索“抓—放—再抓—插”的模式序列，再逐层加入关键位姿、整段几何和轨迹约束，让连续优化尽早淘汰坏序列。它需要模式切换、几何关系、代价和约束接口；失败时要把对应序列、违反的约束和残差送回搜索层，局部优化失败不能被误说成物理世界绝无方案。Toussaint 与 Lopes 的逻辑—几何规划研究展示了这种边界。

两者都没有取消自由空间、动力学和接触规划，只是改变了离散选择怎样向它们提问、连续失败怎样返回。完整领域语法、求解器实现和规模比较留给科研工具书；主线先把输入、返回语义和停止条件讲清楚。

一个动作不只有成功和失败

计划交给执行器以后，每个动作还要区分：前置条件不满足、运行中、成功、已知失败、超时、被中止和未知。“函数已经返回”只是软件事件，不等于动作的后置条件已经得到证据。

插入卡滞后退回，维修座、孔口接触、视觉特征和柔顺控制的内部状态都可能改变。恢复必须重建受影响的感知、抓取、连续计划与缓存；重复调用而不改变失败原因，只会更稳定地重现失败。人工接管也要单独记录：零件后来合格，不等于自动流程独立完成。

成功必须是可以复查的合取

最终的“完成”不能由最后一个节点自报。对这次维修座，成功谓词要求深度、姿态、作用、量规、停止事件和数据完整性同时成立：

例如深度要落在 31.5–32.5 mm31.5\text{--}32.5\ \mathrm{mm}31.5–32.5 mm，倾斜不超过 0.5∘0.5^\circ0.5∘，轴向与侧向作用都不越界，量规通过，也没有未处理的碰撞、滑移、停止或缺失记录。任何必要证据未知，结果就仍是未知，不能被其他真值盖过去。

我终于得到一条完整候选：当前抓法搬离夹具，在中间台面放稳，松手后从另一侧再抓，重新检查自由空间与接触运动，最后插入并通过成功谓词。老师傅看完却用三条彼此不同的路径也做到了同一结果。下一章的问题因此出现：如果合格不要求动作一模一样，我究竟该从他的示范里学什么？

练习

五题都发生在我为同一维修座寻找“放稳—换边再抓—插入”路线的过程中。先决定我应保留哪一项结果、让哪一层重算或停在哪里，再说明专业原因。

练习 1：路能绕开，手腕方向却仍不对

我保留的当前抓法在所有无碰路线中都会让手腕在插入前碰护栏；另一抓法可以对准孔轴。搜索器却不断给当前抓法换路。我应先改哪一层，并怎样记录这次失败？

练习 2：能不能先松手，再去寻找支撑

我想把“松手”移到“稳定放置”之前，以减少一个检查。维修座下方有两个台面：一个能承重但挡住另一侧靠近，另一个承重、可见且可再抓。应怎样安排动作，为什么不能只做顺序交换？

练习 3：身体加了护套，旧检查还能不能用

我的手腕新增了一层防护软垫，维修座、目标和台面没有改变。缓存里保存着旧身体下三条可行边和五条碰撞边。哪些记录可以直接沿用，哪些必须重新检查？

练习 4：没找到方案，还是还看不清

我留下两种停止记录。情形甲：冻结的抓法、支撑位和模式候选已全部检查，均因稳定性或碰撞失败。情形乙：唯一中间台面是否被工具占用无法观测。两种情形应怎样停止，下一步分别是什么？

练习 5：搜索到了“已插入”节点，就算完成了吗

我的再抓取和插入都通过连续检查，最终深度为 30.9 mm30.9\ \mathrm{mm}30.9 mm，量规未通过；各动作节点均返回成功。维修座和自动流程应怎样标记？
