计划说脚已经落稳，我为什么还在滑——意外接触怎样触发恢复

预测让我避开了几步以后的死路，却没有让世界按预测行事。

一只摆动脚还没有到计划中的落地时刻，脚尖先碰到碎石边缘。法向作用估计突然跳到 9 N9\,\mathrm N9N，下一份样本又落回 3 N3\,\mathrm N3N。我刚想把它记成提前触地，信号便消失了。

稍后，作用估计连续升到 10 N10\,\mathrm N10N、11 N11\,\mathrm N11N 和 12 N12\,\mathrm N12N。这一次，石块真的开始承载，却也轻轻转了一下，原来的地图位置随之失效。

我让另一个已经确认接触的脚继续支撑。它的接触标记始终为真，足端相对地面的位置却沿潮湿接缝缓慢移动。

“计划说第一只脚还没落地，信号却说碰到了；第二只脚明明还在接触，为什么又会滑？”我问博士，“一个真或假的接触标记，究竟能保证什么？”

博士把三个问题分开：

  1. 计划希望哪只脚在什么时刻接触；
  2. 当前记录是否足以确认法向接触真的发生；
  3. 已接触的脚是否还能按‘相对地面不动’来承担支撑。

第一项是意图，第二项是触地事件，第三项才涉及支撑质量。它们不能共用一个布尔值。

这些作用仍来自原有记录，不是新装了一只力传感器

我的四足身体没有新增部件。法向与切向作用来自第二幕已经建立的驱动、动力学、状态与接触估计之间的关系；足端相对运动来自既有关节运动学、状态估计和局部地形记录。

因此，下面的“作用估计”不是足底新传感器的直接读数。它会受到模型参数、驱动估计、接触位置和状态误差共同影响。正因为来源并不完美，单个尖峰不能立刻变成可靠触地结论。

接触记录不再只有真假，而是至少包含：

  • 未接触：当前没有确认接触；
  • 待确认：出现了接触候选，但连续证据还不够；
  • 已确认：已经满足连续样本条件；
  • 候选首次出现、正式确认与解除的时刻；
  • 触发原因、样本来源和接触脚身份；
  • 相对计划时刻属于提前、按时、延迟还是未知。

接触状态机怎样处理尖峰、迟滞带和超时

用自然语言写出状态变化：

未接触：
  f_n >= f_on
    → 待确认，连续接通计数设为 1，保存候选开始时刻
  否则
    → 保持未接触

待确认：
  f_n >= f_on
    → 连续接通计数加 1；达到 3 时转为已确认
  f_n <= f_off
    → 候选中断，回到未接触
  f_off < f_n < f_on
    → 仍是待确认，但“连续高于接通阈值”的计数清零
  候选状态超过声明时限

已确认：
  f_n <= f_off
    → 连续断开计数加 1；达到 3 时转为未接触

确认时刻取第三份连续高样本的时间；候选开始时刻仍另外保存。两者不能合并，否则回放时无法知道系统花了多久才获得足够证据。

若计划落地时刻已到，却在允许延迟窗口内仍没有确认接触，就记录延迟触地或无支撑候选，并保持其他脚的支撑；不能因为计划表已经翻到“支撑期”就伪造接触。

不同单位的危险线索不能加成一个总分

这次事件同时出现：

  • 法向接触作用，单位 N\mathrm NN；
  • 切向速度，单位 m/s\mathrm{m/s}m/s；
  • 相对位移与误差管，单位 m\mathrm mm；
  • 摩擦余量，单位 N\mathrm NN；
  • 外部作用残差的力分量与力矩分量，单位 N\mathrm NN 与 N m\mathrm{N\,m}Nm；

恢复空间、接触或停止信息不足 停止并求助 不进行无边界恢复

事件发生后，旧计划怎样失去执行权

输入：
  计划接触时刻与当前预测第一项
  带时间的法向/切向作用估计
  足端相对速度、位移、状态估计与局部地图
  当前全身控制、驱动、停止和管束假设

1. 按时间更新每只脚的“未接触／待确认／已确认”状态
2. 记录候选、确认、解除以及相对计划的提前/延迟时刻
3. 接触事件与计划不一致时，立即取消旧计划后续项的执行权
4. 对已确认的接触检查切向速度、位移和保守摩擦余量
5. 滑移候选满足连续证据时，取消固定足端约束并更新可用支撑
6. 用质心动量残差记录模型未解释部分，但不直接命名其来源
7. 若石块移动，更新地图；旧网格和旧落点保持失效
8. 核对扰动是否仍在声明集合内；否则不再使用旧误差管保证
9. 分别检查接触、摩擦、几何、误差管、驱动、时间和停止余量
10. 选择低速继续、改变落脚、扩大支撑、退回、停止或求助
11. 恢复动作重新经过预测、全身控制和驱动边界，不享有例外
12. 只发送新候选第一项；持续记录实际结果并准备再次停止

接触与滑移状态机对每份样本只做常数规模更新；长度为 TTT 的回放需要 O(T)O(T)O(T) 时间。质心残差和标量管束算例也可逐步更新。真实鲁棒 MPC 集合运算与多维不变集计算会复杂得多，当前屏幕记录不包含这些运算的实现或计时。

我退回的不是起点，而是一条已经失效的解释

面对潮湿接缝，我没有继续沿旧计划把身体向前推。提前触地让原接触时序失效，石块转动让原地图网格失效；确认滑移又让“足端固定”这条约束失效。

我先保持仍可靠的支撑，减小切向需求，再退回一只脚到已知干燥落点，扩大支撑范围。博士随后重新测量移动过的石块，并把旧落点标成不可复用。

这次退回不是失败被擦掉，而是恢复动作的一部分。但屏幕上只有用于解释恢复顺序的记录，没有真实硬件记录证明身体完成了同样的退步、扩大支撑或停止。

现在我已经能把提前触地、滑移、模型外残差和恢复决定按时间留下。最后还缺一件事：当我最终绕过移动的石块并走出通道时，怎样让“到达”不把这些中途发生的事全部抹平。
