模型说没有碰撞，为什么手肘仍擦到工具架——配置空间与碰撞距离

我让底盘保持锁定，把上一章的轨迹送到工具架入口前做最低速的受控检查。屏幕上的 TCP 线始终离架子很远，离散采样点也全部显示绿色。

可手臂经过中段时，肘部保护垫还是轻轻擦过了架沿。我立刻停住。回看数字身体，肘部只用一根偏细的圆柱代替；轨迹检查还只看了每隔一段时间的关节姿态。

我问：“明明手没有碰到，模型也说没有碰撞，为什么身体还是碰到了？”

博士先让我指出“模型看的是哪一部分身体、哪两个时刻、哪一版世界”。直到这些问题分开，他才把整条机械臂在关节空间中造成的禁区称为配置空间障碍，把用于快速查询的简化外形称为碰撞代理，把两个采样时刻之间也纳入判断的办法称为连续碰撞检测。

环境碰撞和自碰必须分开列出

环境碰撞检查每个连杆与工具架、台面、墙体等场景对象。自碰则检查身体内部不应相交的连杆对：

相邻连杆在关节处本来就共享或重叠一部分几何，不能一律当成碰撞；但也不能为了消除误报，把整条手臂的相邻若干对全部忽略。允许相交的对象对，应由结构与用途明确登记。

本章身体仍是锁定底盘的移动操作形态。轮足、机身、机械臂与张开的夹爪都要进入环境碰撞；不能只加载新装的手臂模型。身体版本变化时，旧碰撞对象对与代理也随之失效。

障碍膨胀把有体积的身体变成禁区

对只做平移的刚体，机器人形状 RRR 与障碍 OOO 相交，可以借助闵可夫斯基和写成点的配置障碍：

Cobs=O⊕(−R).\mathcal C_{\mathrm{obs}} = O\oplus(-R).Cobs​=O⊕(−R).

直观上，是把障碍向外“长出”一层机器人外形，再让机器人参考点绕开它。

关节机械臂的形状会随 qqq 改变，不能只把工作空间障碍统一膨胀一次就得到完整高维配置障碍。但这个思想仍提醒我：规划的不是无尺寸的 TCP，而是由当前身体姿态决定的整组体积。

增加安全外扩量，也可视为让碰撞代理或障碍再膨胀一层。外扩量的来源必须说明：它可能包含几何简化、标定范围、场景测量误差和希望保留的恢复余量，而不是一个没有来源的“大概安全距离”。

碰撞代理为什么既要简单又不能太瘦

渲染网格用于看起来像真实身体，碰撞网格用于高频几何查询。常见代理包括：

  • 球、胶囊、圆柱与长方体；
  • 凸包与多个凸体的组合；
  • 简化三角网格；
  • 对环境预计算的距离场。

简单代理查询快，却必须说明它是外包络、内近似还是混合近似：

  • 外包络覆盖真实形状，较保守，可能错过狭窄但真实可通过的路；
  • 内近似比真实形状瘦，可能把真实碰撞误报成自由；
  • 局部混合代理需要逐连杆记录误差范围。

本次肘部圆柱比保护垫半径小，属于危险的内近似。仅把查询频率加倍，仍会反复检查同一个错误形状。

碰撞代理要绑定当前数字身体的连杆、局部坐标系、尺寸、版本和适用配置范围。网格缺面、缩放单位错误、局部原点错位或工具模型缺失时，应输出“模型无效”，而不是把“没有三角形相交”写成安全。

广相先排除不可能相遇的对象

若 LLL 个身体连杆与 MMM 个场景对象逐对做精确网格检查，成本会迅速增加。碰撞系统通常先做粗筛阶段：

  1. 为每个对象计算 AABB、OBB、包围球等粗包围体；
  2. 用层次包围体树 BVH 组织网格或对象；
  3. 排除包围体明显分离的对象对；
  4. 把剩余少量候选交给精查阶段。

粗筛输出“可能相交的候选对”，不是“已经碰撞”。同样，被正确排除只证明当前包围体在当前配置下分离；若变换或时间版本错误，树再快也没有意义。

凸体距离与穿透方向分别怎样求——GJK 与 EPA

对两个凸集 A,BA,BA,B，它们相交等价于原点是否属于闵可夫斯基差：

A⊖B={a−b:a∈A,b∈B}.A\ominus B = \{a-b:a\in A,b\in B\}.A⊖B={a−b:a∈A,b∈B}.

GJK 使用支撑映射逐步构造单纯形，寻找这个差集上离原点最近的点。它可用于：

  • 判断两个凸体是否分离；
  • 在分离时估计距离与最近点；
  • 在原点被包含时判断相交。

非凸网格通常要先做凸分解，或在 BVH 叶节点上处理三角形对。

GJK 检出相交后，普通距离已经是零，不能直接说明往哪个方向退出、穿入多深。EPA 从包含原点的多面体向外扩展，近似给出穿透深度和方向。博士提醒我：

  • GJK/EPA 都依赖数值容差与终止条件；
  • 迭代超限或非有限值应返回“未知/数值失败”；
  • EPA 的方向是当前代理几何的局部结果，不是接触力方向证明；
  • 穿透结果不能被用来授权继续挤压。

Gilbert、Johnson 与 Keerthi 1988 年的原始论文给出了经典凸体距离过程；本章只迁移支撑映射、分离/相交与最近点责任。

对轨迹采样

简单改进是沿连接相邻配置的规划边自适应细分：

  1. 按关节最大变化或几何位移界估计步长；
  2. 检查中点；
  3. 净空小或几何变化大时继续二分；
  4. 达到分辨率上限仍无法证明时返回未知。

改善办法有两类：

  • 增大名义净空：换路径、换 IK 姿态、重新定位身体；
  • 缩小有依据的不确定范围：细化代理、重测场景、更新标定、验证柔性边界。

不能只把界面上的阈值从 12 mm 改成 6 mm。

  • 摩擦系数没有经过辨识，不能支持夹持；
  • EPA 穿透方向不是接触力；
  • 允许某个对象对在未来抓取时接触，不等于现在可以碰；
  • 抓取中的手指—毛坯碰撞语义要等接触模型与成功条件建立。

碰撞模型怎样裁决一条连续候选

输入：
  当前数字身体
  当前工坊场景
  当前自由空间运动计划
  允许相交的自碰对象对、代理外扩量与不确定范围

1. 核对身体、标定、场景、轨迹和几何版本。
2. 验证代理完整性、单位、局部坐标系、缩放和对象对列表。
3. 对每个离散状态：
   a. 更新所有连杆变换；
   b. 粗筛阶段生成候选对象对；
   c. 用 GJK、网格或距离场做精查；
   d. 保存最小距离、见证对象对与求解状态。
4. 对每条相邻规划边：
   a. 估计扫掠范围；
   b. 自适应细分或执行适用的 CCD；
   c. 保存首次接触参数或连续分离证据。
5. 合并名义距离、保守不确定范围与求解残差。
6. 输出：
   安全      -> 全部硬边界通过；
   边界不足  -> 数值有限但余量不足，不执行；
   碰撞      -> 有符号距离 <= 0；
   未知      -> 场景/代理缺失、数值失败或连续检查不足。

输出碰撞模型：
  几何身份与版本
  自碰与环境对象对策略
  名义距离与不确定范围的构造
  离散/连续方法及其分辨率
  第一个见证对象对与状态

“安全”只对冻结的身体、场景和轨迹成立。场景移动、工具更换、标定更新或轨迹重新定时后，都要按依赖关系重新判断。
