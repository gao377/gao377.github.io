模型说没有碰撞，为什么手肘仍擦到工具架——配置空间与碰撞距离

我让底盘保持锁定，把上一章的轨迹送到工具架入口前做最低速的受控检查。屏幕上的 TCP 线始终离架子很远，离散采样点也全部显示绿色。

可手臂经过中段时，肘部保护垫还是轻轻擦过了架沿。我立刻停住。回看数字身体，肘部只用一根偏细的圆柱代替；轨迹检查还只看了每隔一段时间的关节姿态。

我问：“明明手没有碰到，模型也说没有碰撞，为什么身体还是碰到了？”

博士先让我指出“模型看的是哪一部分身体、哪两个时刻、哪一版世界”。直到这些问题分开，他才把整条机械臂在关节空间中造成的禁区称为配置空间障碍，把用于快速查询的简化外形称为碰撞代理，把两个采样时刻之间也纳入判断的办法称为连续碰撞检测。

工具中心只是身体上的一个点

机械臂配置记为

q∈Q.q\in\mathcal Q.q∈Q.

在配置 qqq 下，整条身体占据工作空间集合

R(q)=⋃ℓ∈连杆Rℓ(q).\mathcal R(q) = \bigcup_{\ell\in\text{连杆}}\mathcal R_\ell(q).R(q)=ℓ∈连杆⋃​Rℓ​(q).

世界障碍集合记为 O\mathcal OO。发生环境碰撞的配置集合为

Cobsworld={q∈Q:R(q)∩O≠∅}.\mathcal C_{\mathrm{obs}}^{\mathrm{world}} = \left\{ q\in\mathcal Q: \mathcal R(q)\cap\mathcal O\ne\varnothing \right\}.Cobsworld​={q∈Q:R(q)∩O=∅}.

自由配置空间为

Cfree=Q∖(Cobsworld∪Cobsself).\mathcal C_{\mathrm{free}} = \mathcal Q \setminus \left( \mathcal C_{\mathrm{obs}}^{\mathrm{world}} \cup \mathcal C_{\mathrm{obs}}^{\mathrm{self}} \right).Cfree​=Q∖(Cobsworld​∪Cobsself​).

博士把以关节配置为坐标、把碰撞配置视为障碍的描述称为配置空间。

同一个 TCP 位姿可能对应肘上、肘下或不同腕部姿态；它们在工作空间占据的整臂形状不同。因此，“末端点无碰”既不能推出手肘无碰，也不能推出另一组 IK 解无碰。

环境碰撞和自碰必须分开列出

环境碰撞检查每个连杆与工具架、台面、墙体等场景对象。自碰则检查身体内部不应相交的连杆对：

Ri(q)∩Rj(q).\mathcal R_i(q)\cap\mathcal R_j(q).Ri​(q)∩Rj​(q).

相邻连杆在关节处本来就共享或重叠一部分几何，不能一律当成碰撞；但也不能为了消除误报，把整条手臂的相邻若干对全部忽略。允许相交的对象对，应由结构与用途明确登记。

本章身体仍是锁定底盘的移动操作形态。轮足、机身、机械臂与张开的夹爪都要进入环境碰撞；不能只加载新装的手臂模型。身体版本变化时，旧碰撞对象对与代理也随之失效。

障碍膨胀把有体积的身体变成禁区

对只做平移的刚体，机器人形状 RRR 与障碍 OOO 相交，可以借助闵可夫斯基和写成点的配置障碍：

Cobs=O⊕(−R).\mathcal C_{\mathrm{obs}} = O\oplus(-R).Cobs​=O⊕(−R).

直观上，是把障碍向外“长出”一层机器人外形，再让机器人参考点绕开它。

关节机械臂的形状会随 qqq 改变，不能只把工作空间障碍统一膨胀一次就得到完整高维配置障碍。但这个思想仍提醒我：规划的不是无尺寸的 TCP，而是由当前身体姿态决定的整组体积。

增加安全外扩量，也可视为让碰撞代理或障碍再膨胀一层。外扩量的来源必须说明：它可能包含几何简化、标定范围、场景测量误差和希望保留的恢复余量，而不是一个没有来源的“大概安全距离”。

碰撞代理为什么既要简单又不能太瘦

渲染网格用于看起来像真实身体，碰撞网格用于高频几何查询。常见代理包括：

  • 球、胶囊、圆柱与长方体；
  • 凸包与多个凸体的组合；
  • 简化三角网格；
  • 对环境预计算的距离场。

简单代理查询快，却必须说明它是外包络、内近似还是混合近似：

  • 外包络覆盖真实形状，较保守，可能错过狭窄但真实可通过的路；
  • 内近似比真实形状瘦，可能把真实碰撞误报成自由；
  • 局部混合代理需要逐连杆记录误差范围。

本次肘部圆柱比保护垫半径小，属于危险的内近似。仅把查询频率加倍，仍会反复检查同一个错误形状。

碰撞代理要绑定当前数字身体的连杆、局部坐标系、尺寸、版本和适用配置范围。网格缺面、缩放单位错误、局部原点错位或工具模型缺失时，应输出“模型无效”，而不是把“没有三角形相交”写成安全。

广相先排除不可能相遇的对象

若 LLL 个身体连杆与 MMM 个场景对象逐对做精确网格检查，成本会迅速增加。碰撞系统通常先做粗筛阶段：

  1. 为每个对象计算 AABB、OBB、包围球等粗包围体；
  2. 用层次包围体树 BVH 组织网格或对象；
  3. 排除包围体明显分离的对象对；
  4. 把剩余少量候选交给精查阶段。

粗筛输出“可能相交的候选对”，不是“已经碰撞”。同样，被正确排除只证明当前包围体在当前配置下分离；若变换或时间版本错误，树再快也没有意义。

凸体距离与穿透方向分别怎样求——GJK 与 EPA

对两个凸集 A,BA,BA,B，它们相交等价于原点是否属于闵可夫斯基差：

A⊖B={a−b:a∈A,b∈B}.A\ominus B = \{a-b:a\in A,b\in B\}.A⊖B={a−b:a∈A,b∈B}.

GJK 使用支撑映射逐步构造单纯形，寻找这个差集上离原点最近的点。它可用于：

  • 判断两个凸体是否分离；
  • 在分离时估计距离与最近点；
  • 在原点被包含时判断相交。

非凸网格通常要先做凸分解，或在 BVH 叶节点上处理三角形对。

GJK 检出相交后，普通距离已经是零，不能直接说明往哪个方向退出、穿入多深。EPA 从包含原点的多面体向外扩展，近似给出穿透深度和方向。博士提醒我：

  • GJK/EPA 都依赖数值容差与终止条件；
  • 迭代超限或非有限值应返回“未知/数值失败”；
  • EPA 的方向是当前代理几何的局部结果，不是接触力方向证明；
  • 穿透结果不能被用来授权继续挤压。

Gilbert、Johnson 与 Keerthi 1988 年的原始论文给出了经典凸体距离过程；本章只迁移支撑映射、分离/相交与最近点责任。

有符号距离与净空要带见证对象

对一对几何体，可定义有符号距离：

d(A,B){>0,分离,=0,接触边界,<0,代理穿透.d(A,B) \begin{cases} >0,&\text{分离},\\ =0,&\text{接触边界},\\ <0,&\text{代理穿透}. \end{cases}d(A,B)⎩⎨⎧​>0,=0,<0,​分离,接触边界,代理穿透.​

一组配置的最小净空为

dmin⁡(q)=min⁡(i,j)∈Pd ⁣(Ri(q),Oj),d_{\min}(q) = \min_{(i,j)\in\mathcal P} d\!\left(\mathcal R_i(q),\mathcal O_j\right),dmin​(q)=(i,j)∈Pmin​d(Ri​(q),Oj​),

并另行纳入自碰对象对。

只保存最小数字不够。还要保存：

  • 哪个连杆与哪个对象构成最小距离对象对；
  • 两侧最近点及其表达坐标系；
  • 查询时刻与几何版本；
  • 求解残差和终止状态；
  • 该距离是否来自网格、凸体还是距离场。

距离场能快速查询空间点到障碍表面的距离及局部梯度，但把一个连杆只采成少量表面点，仍可能漏掉两点之间的几何。场的分辨率、截断范围、符号定义和未知区域都属于证据。

离散状态无碰，不等于两点之间无碰

对轨迹采样

q0,q1,…,qN,q_0,q_1,\ldots,q_N,q0​,q1​,…,qN​,

离散检查只验证每个 qkq_kqk​。若手肘在 qkq_kqk​ 与 qk+1q_{k+1}qk+1​ 之间扫过架沿，两端都可能分离。

简单改进是沿连接相邻配置的规划边自适应细分：

  1. 按关节最大变化或几何位移界估计步长；
  2. 检查中点；
  3. 净空小或几何变化大时继续二分；
  4. 达到分辨率上限仍无法证明时返回未知。

更直接的连续碰撞检测（CCD）考虑

q(α),α∈[0,1],q(\alpha),\qquad \alpha\in[0,1],q(α),α∈[0,1],

所产生的扫掠体，求是否存在首次接触参数 α⋆\alpha^\starα⋆。保守推进利用当前距离和相对运动上界逐步前进，避免跨过首次接触；对多面体、关节运动和数值误差仍需明确适用条件。

Tang 等人的 C2A 连续碰撞检测论文讨论了受控保守推进；本章不声称任何一个 CCD 实现已经覆盖当前七关节代理，只保留“整条规划边需要证据”的方法边界。

八毫米余量为什么仍然不能执行

当前候选在名义模型上的最小净空为

dnom=8 mm.d_{\mathrm{nom}}=8\ \mathrm{mm}.dnom​=8 mm.

身体代理、标定、场景测量和柔性合成后的保守不确定范围为

ucons=12 mm.u_{\mathrm{cons}}=12\ \mathrm{mm}.ucons​=12 mm.

若把这些量都解释为同一法向上的保守范围，则可保证的剩余量最多为

dcert=dnom−ucons=8−12=−4 mm.d_{\mathrm{cert}} = d_{\mathrm{nom}}-u_{\mathrm{cons}} =8-12 =-4\ \mathrm{mm}.dcert​=dnom​−ucons​=8−12=−4 mm.

结论不是“只差四毫米，可以慢一点”，而是当前证据无法排除真实碰撞，所以不能执行。

这 12 mm 是已经冻结的合成范围，不能再把其中的分项重复相加。若分项只是概率标准差，也不能未经独立性与分布假设就直接当成确定范围；若安全判断需要确定边界，就应提供相应置信或保守构造。

改善办法有两类：

  • 增大名义净空：换路径、换 IK 姿态、重新定位身体；
  • 缩小有依据的不确定范围：细化代理、重测场景、更新标定、验证柔性边界。

不能只把界面上的阈值从 12 mm 改成 6 mm。

软碰撞代价不能抵消硬停止条件

轨迹优化常用平滑代价鼓励远离障碍，例如

ϕ(d)={12(dsoft−d)2,d<dsoft,0,d≥dsoft.\phi(d)= \begin{cases} \frac12(d_{\mathrm{soft}}-d)^2,&d<d_{\mathrm{soft}},\\ 0,&d\ge d_{\mathrm{soft}}. \end{cases}ϕ(d)={21​(dsoft​−d)2,0,​d<dsoft​,d≥dsoft​.​

它适合在多个可行候选间比较余量。真正的硬边界仍应单独检查：

dnom−ucons≥drequired.d_{\mathrm{nom}}-u_{\mathrm{cons}} \ge d_{\mathrm{required}}.dnom​−ucons​≥drequired​.

降低碰撞代价权重可能让总目标更小，却不能让负净空变得合法。求解残差也要计入边界：如果距离查询或优化约束只满足到 ε\varepsilonε，实际要求应为

d≥drequired+ucons+ε.d\ge d_{\mathrm{required}} +u_{\mathrm{cons}} +\varepsilon.d≥drequired​+ucons​+ε.

恢复余量与触发余量还可以不同，形成迟滞：一旦进入边界状态，必须退到更宽距离才恢复，而不是在阈值附近反复切换。

摩擦参数可以登记，但本章不让它们替接触作证

数字身体可以为未来保存材料、摩擦系数范围、表面顺应性和允许接触的对象对。可本章的目标是自由空间无碰：

  • 摩擦系数没有经过辨识，不能支持夹持；
  • EPA 穿透方向不是接触力；
  • 允许某个对象对在未来抓取时接触，不等于现在可以碰；
  • 抓取中的手指—毛坯碰撞语义要等接触模型与成功条件建立。

因此，夹爪仍张开，毛坯仍由夹具台支撑。

碰撞模型怎样裁决一条连续候选

输入：
  当前数字身体
  当前工坊场景
  当前自由空间运动计划
  允许相交的自碰对象对、代理外扩量与不确定范围

1. 核对身体、标定、场景、轨迹和几何版本。
2. 验证代理完整性、单位、局部坐标系、缩放和对象对列表。
3. 对每个离散状态：
   a. 更新所有连杆变换；
   b. 粗筛阶段生成候选对象对；
   c. 用 GJK、网格或距离场做精查；
   d. 保存最小距离、见证对象对与求解状态。
4. 对每条相邻规划边：
   a. 估计扫掠范围；
   b. 自适应细分或执行适用的 CCD；
   c. 保存首次接触参数或连续分离证据。
5. 合并名义距离、保守不确定范围与求解残差。
6. 输出：
   安全      -> 全部硬边界通过；
   边界不足  -> 数值有限但余量不足，不执行；
   碰撞      -> 有符号距离 <= 0；
   未知      -> 场景/代理缺失、数值失败或连续检查不足。

输出碰撞模型：
  几何身份与版本
  自碰与环境对象对策略
  名义距离与不确定范围的构造
  离散/连续方法及其分辨率
  第一个见证对象对与状态

“安全”只对冻结的身体、场景和轨迹成立。场景移动、工具更换、标定更新或轨迹重新定时后，都要按依赖关系重新判断。

世界没有只碰到我的手，它碰到的是整个我

肘部擦痕没有证明碰撞算法“没用”，它证明我问错了对象，也使用了过瘦的身体。把 TCP 换成整条运动链，把离散点换成连续规划边，再把不确定范围从名义距离中扣除，我才有依据说一条候选当前可用或不可用。

现在我能排除穿过架子的直线，却还不知道该从上方还是侧方绕过去。配置空间有七个维度，无法把每个角落都列成表。下一章要让有限采样在这个巨大空间里寻找一条可复核的绕行路线。

练习

四题都发生在狭窄工具架入口。先决定继续检查、改模型、改路还是停止，再说明碰撞术语；不把 TCP、采样点或名义网格当成整段真实身体。

练习 1：TCP 离架子五厘米，谁还可能碰到

整段 TCP 采样与工具架最小距离为 50 mm50\ \mathrm{mm}50 mm；第 4 连杆的肘部代理与架沿距离却为 −1 mm-1\ \mathrm{mm}−1 mm。程序只把 TCP 距离写进计划摘要。

决定候选能否继续。说明配置空间判断要加入哪些身体部件与对象对，再选一种不依赖 TCP 距离的检查办法核对结论。

练习 2：两个绿色采样点之间藏着一次碰撞

相邻配置 qk,qk+1q_k,q_{k+1}qk​,qk+1​ 的离散净空都是 15 mm15\ \mathrm{mm}15 mm，中间扫掠检查在 α=0.46\alpha=0.46α=0.46 发现肘部有符号距离为 −2 mm-2\ \mathrm{mm}−2 mm。

决定加密显示采样、缩短时间还是舍弃这条规划边。说明 CCD 的结果为何不能被两个端点抵消。

练习 3：名义八毫米为何不是“基本够了”

一条候选的名义最小净空为 8 mm，保守合成不确定范围为 12 mm。计算可保证剩余量，并在“减小阈值、细化模型、重标定、改路、最低速执行”中选择合理动作。

练习 4：代理越简单，结果越快，能否直接采用

肘部真实保护垫外径比代理大 5 mm；一块工具的场景网格版本未知；GJK 对其余凸体均正常返回正距离。系统总耗时很低。

判断结果应为“安全”“边界不足”“碰撞”还是“未知”。说明哪些修复用于恢复几何有效性，哪些用于增加路径余量，并指出摩擦参数为什么帮不上忙。
