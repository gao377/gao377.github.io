第13幕

界面写着“就绪”，泵为什么仍没有转——从请求到物理结果的联锁链

设备间的绿色指示写着“就绪（ready）”。旁边还能找到一条“启动请求已接受（start accepted）”的旧记录。可主泵没有转，管道里也没有出现能够和启动对应的持续变化。

我低头看向第九幕装上的替换部件。它的机械锁止标记仍在，最容易讲出的故事是“新部件坏了”。这个故事太完整了，完整得没有给其他未知留下位置：远端闸门仍然没有结果，电气隔离记录已经过期，电机电流没有新的可用样本，当前共同任务也根本没有授权再次启动主泵。

我想起系统第一次把“命令已经送达”误写成“设备已经动作”时的情形。同样一句“就绪”，可能只属于界面、控制器、机械安装或人员流程中的一层。泵不转之前，必须先问清究竟哪一层曾经说过什么。

从意图走到水位变化，中间没有可以省略的一步

我沿时间把事件重新排开：

共同任务仍需要这个动作吗
→ 当前权限允许谁请求哪一台泵
→ 人员、隔离、阀位和机械条件是否闭合
→ 启动请求是否真正发出
→ 设备是否接收并接受
→ 联锁是否允许作用施加
→ 电机是否出现可观察电流与转动
→ 泵和管路是否产生可观察响应
→ 水位是否在声明的时间窗里改变

这条链里，前一项成立只是后一项的必要条件之一。“就绪”若只表示界面进程在线，就不能替代机械就绪；机械锁止正确，也不能替代电气状态和闸门位置；“已接受”只说明请求进入了某个处理流程，不能替代“已施加”或物理结果。

我把眼前能说和不能说的内容分开：

层次 当前记录 能得出的结论
共同任务与权限 当前任务只允许检查主泵，不允许重启 新启动动作不得发出
远端水力条件 闸门结果 UNKNOWN 不能假设管路已经处于启动条件
机械条件 替换部件锁止可见 只支持这一项局部机械检查
电气条件 隔离样本过期，电流新样本缺失 不能判断电气链是否允许或是否施加
设备作用 旧记录写着“已接受” 不能推出电机已得到作用
物理结果 没有可信转动、流量或水位响应链 主泵结果保持 UNKNOWN，设备继续隔离

诊断不是在未知中挑一个最像原因的词

我截取从旧启动请求到当前时刻的相关事件，这就是此前学过的因果切片。切片不只收集故障附近的日志，还要保留每条记录的来源、时钟、动作编号、权限轮次和失效条件。

眼前至少有几种仍然相容的解释：

  • 请求被界面接受，却没有进入允许施加的控制层；
  • 远端闸门状态使联锁保持断开；
  • 电气隔离或供电状态与旧记录不同；
  • 电机动作发生过，但当前缺少能确认它的观测；
  • 新部件或其他机械条件存在问题。

这些解释不能靠故事的顺畅程度排序。要区分它们，需要彼此独立的检查：让维修人员复核电气隔离，让远端或现场人员确认闸门，让本地传感记录电流和转动，再把管路响应与水位时刻对齐。当前我没有这些完整结果，所以不能把故障归到新部件，也不能把“尚未证明故障”反过来写成“部件正常”。

能触及面板，不等于有权把诊断变成动作

人形身体让我能够站在面板前、看见锁止、触及手柄，也能在狭小设备间保持工具和视线。可是身体可达性只解决“我能不能碰到”，不解决“我是否获准碰”“系统是否准备好”以及“碰了以后怎样知道结果”。

我保持主泵隔离，把它记录成待人员继续复核的对象。随后转向本地可以逐项确认的备用链路：它有独立阀路和本地面板，不依赖把远端闸门结果猜成成功。但备用手轮旁仍有维修人员，潮湿地面也使我的足底接触余量不再确定。

练习

练习 1：绿色的“就绪”到底允许我说什么

界面显示“就绪”，旧请求显示“已接受”；远端闸门结果未知，电气隔离样本过期，新的电机电流、转速和流量记录都缺失。请列出当前可以确认的层次、仍然未知的层次和必须停止的动作。

练习 2：新部件安装后泵没转，能否判它故障

替换部件的机械锁止可见，但同一时间窗内还有权限、闸门、电气和观测缺口。请给出至少三种替代解释，安排能区分它们的检查顺序，并说明为什么诊断结果不能自动触发重启。

练习 3：怎样重建一段没有完整时钟的事件链

界面、控制器、电气记录和水位记录来自不同设备，部分只有接收时刻。请说明因果切片应保存哪些身份与时间字段，哪些事件只能保持并列，以及怎样避免把晚到消息排成晚发生事件。

参考答案

主泵的沉默没有被我压成一个轻率的故障标签。另一条路仍然存在，但它首先要求我面对一项比模型更直接的事实：手柄附近还有人。
