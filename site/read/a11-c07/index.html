第十一幕

清洗后的数据，怎样找到它原来的样子——数据登记、版本与谱系

同一段 raw 经过两种 alignment rule，得到两个看起来都合理的 derived sample。后来，其中一个依赖的 calibration 被撤销，可用它训练过的候选仍在登记表里显示 active。

我试着打开“最新版”文件，却发现文件名既没告诉我它来自哪条 raw，也没告诉我哪些下游还欠一次重算。

博士把这条从原始记录、转换、数据集、拆分直到候选模型的关系称为 数据谱系。

版本不是覆盖旧文件，而是生成新身份

每一种对象都需要稳定 ID、revision 和 hash：

原始记录
  └─ 转换（代码、配置、版本）
       └─ 派生经历或样本
            └─ 数据集版本
                 └─ 数据划分版本
                      └─ 模型或候选版本
                           └─ 证据或发布记录

派生对象只读引用 parent ID/hash 与 transform code/config hash。相同 raw 经两种 alignment rule 产生的是两个 sibling revision，不能让后写的文件偷偷改掉先写的含义。

可把谱系写成有向图 G=(V,E)G=(V,E)G=(V,E)。边 u→vu\rightarrow vu→v 表示 vvv 依赖 uuu。若上游 uuu 被撤销或 supersede，所有可达后继

Desc⁡(u)={v∣u⇝v}(1)\operatorname{Desc}(u)=\{v\mid u\leadsto v\} \tag{1}Desc(u)={v∣u⇝v}(1)

都要重新判断，而不是一律删除。

撤销、失效与回滚不是同一个动作

  • superseded：有新 revision 取代旧 revision 的未来用途，旧证据仍保留；
  • revoked：上游身份或依据被判定不可继续使用；
  • stale：下游依赖了已变化上游，尚未重建/复核；
  • invalid-for-use：对某用途硬条件失败；
  • rollback：把活动指针返回一个仍满足依赖的旧 revision；
  • deleted derived cache：可重建的派生缓存被删，不得连带删除 raw 与证据历史。

若 calibration C-7 被撤销，依赖它的 derived samples 先标 stale；包含这些样本的 dataset/split、训练候选和 Benchmark 记录沿图传播。一个候选即使训练已结束，也不能因为产物文件还存在就保持 active。

数据登记也要能发现谱系图本身的错误

数据集登记至少检查：

  • parent 缺失形成 orphan；
  • 内容与登记 hash 不一致；
  • 依赖图出现 cycle；
  • raw/derived、fragment/Episode group 被混作同类；
  • 同源复制跨 split 形成 contamination；
  • 已撤销 calibration/body/map/predicate 仍被 active candidate 使用；
  • retention 或 access 状态与用途冲突。
登记不可变对象身份与内容哈希
登记转换代码、配置、输入和输出哈希
确认父对象存在，并检查谱系没有环
附上数据集归属、数据划分、用途和证据状态

上游对象被替代或撤销时：
    沿谱系遍历所有后代
    将受影响用途标为 STALE 或 INVALID
    阻止这些版本进入新的训练和发布
    生成新版本，或回滚到仍兼容的祖先
保留旧对象和失效原因，供历史证据复核

固定数据集中 12 个 legacy raw fragment、40 个 current Episode group 和 12 个 independent regression group 分开登记。12 raw fragment 不等于 12 Episode group；一个 unsegmentable fragment 更不能因进入 registry 就变成训练样本。

练习

练习 1：同一 raw，两种对齐

rule A 允许的窗口较宽，rule B 遇到 contact reset 就切断。两者生成不同 derived hash。说明 registry 应怎样保存，能否只留分数更高的一份。

练习 2：标定撤销后候选仍 active

calibration C-7 被撤销；dataset D 的三分之一样本依赖它，候选 M 由 D 训练，回归集 R 与 C-7 无关。指出哪些对象 stale、哪些仍可用，以及回滚要满足什么。

练习 3：删除清洗文件会不会删除证据

一个 derived cache 可由 raw 和 transform 确定重建。团队想清理磁盘。判断哪些身份、hash 和历史不能随缓存删除。
