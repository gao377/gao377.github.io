第七幕

栅格里有路，转弯时身体为什么撞墙——非完整约束、状态格与 Hybrid A*

全局路线把我带到侧街入口。栅格中心线先向前，再在墙角处折向右侧；每个格点都离障碍足够远。若身体能像棋子一样原地改变方向，这条线完全正确。

可我记得第四幕的轮子：它们愿意滚动，却拒绝瞬间横移。实际转弯时，身体外侧和背上的部件扫向墙角。

我问博士：“路径上的点都不碰墙，为什么点连起来就碰了？”

博士说，规划不仅要选择经过哪里，还要选择身体怎样在连续时间中改变姿态。端点无碰不保证连接端点的运动存在，也不保证中间扫掠无碰。

他把速度受轮地几何限制、无法任意向所有方向瞬时运动称为非完整约束。

混合 A* 怎样让搜索节点带上朝向

普通栅格只记“我到了哪一格”，却忘了身体以什么朝向到达。对我而言，从东面朝右进入街角和从南面朝上进入同一格，下一步能做的动作完全不同。混合 A* 保留栅格便于搜索的骨架，同时不把连续姿态压成同一个点。

Hybrid A* 在离散网格上管理搜索，却用连续 (x,y,θ)(x,y,\theta)(x,y,θ) 状态和可执行短弧扩展：

  1. 从当前姿态尝试前进或后退的、曲率不同的运动基元；
  2. 对整段基元做碰撞和模式检查；
  3. 将终点映射到离散索引，用于判断节点应当关闭还是重新打开；
  4. 累计长度、倒车、曲率变化和风险代价；
  5. 用满足条件的启发式引导到目标；
  6. 找到候选后再连续化、平滑并复核。

离散朝向过粗会漏路或产生不连续，过细又增加搜索量。解析扩展或 Reeds–Shepp/Dubins 类下界只在其车辆模型和倒车假设下成立。

运动基元把“我能怎样动”预先写进边

状态格规划器预计算一组满足运动模型的短轨迹，再通过旋转和平移把它们放到当前格点。基元可包含：

  • 直行、缓弯、急弯；
  • 前进与受限后退；
  • 轮足滚动与经验证的迈步切换；
  • 进入或退出模式所需的守卫条件（guard）；
  • 时间、扫掠、作用与停止元数据。

基元库太少会把真实存在的路遗漏；库太多会加重搜索。无当前解只说明当前分辨率、基元和预算下未找到，不能自动证明物理无路。

状态格与 Hybrid A* 都让搜索边本身满足运动模型，区别在于怎样保存到达状态：

启发式可以同时使用忽略非完整约束的二维障碍下界，以及在简化车辆模型下的非完整距离下界，再取其中仍可证明不高估的一项组合。启发式只安排展开；完整身体扫掠、湿滑牵引和模式切换仍由真实边检查承担。

我为什么在这里没有重新学习一次 PRM 和 RRT

我想起第五幕为机械臂寻找构型空间路线时，PRM 与 RRT 家族已经教会我一件事：规则格子很难覆盖高维连续空间，随机样本可以把搜索集中到实际连通区域。那段经验仍可用于复杂开放场景，但它不自动给轮足身体一条能转过眼前墙角的路。

当前城市问题有稳定道路方向、频繁重复查询、明确非完整曲率，也需要说清倒车和模式切换各自的代价。状态格或 Hybrid A* 能把这些结构直接写进边和启发式，因此成为本章主方法。采样方法只作为对照：当场景脱离道路结构、状态维数因模式显著增加，而且已有合格的转向函数与局部连接器时，它可能更合适。

公平比较仍要求同一身体、地图、起终状态、连续碰撞检查、时间预算和终端条件。对采样候选再固定 202020 个随机种子，记录是否找到路径、首次得解的时间与路径复核结果；这只检验偶然采样敏感性，不重新推导 PRM 和 RRT，也不预写胜负。概率完备或渐近最优的结论仍只在各自无限样本与连接假设下成立。

选择结果因此不是“格搜索比采样先进”，而是：当前道路结构使状态格和 Hybrid A* 更直接地表达身体怎样转弯；换到高维、弱道路结构的任务时，已有采样经验仍可能重新成为主角。
