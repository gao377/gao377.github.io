一串姿势，为什么还不是一段动作——路径、轨迹与时间参数化

标定更新以后，我重新求出了工具区入口前后的几组关节姿态。起点通过，终点也通过；我便把它们按顺序连起来，以为动作已经完成。

我先只告诉每个关节最后停在哪里。手确实到了终点，中途却在空间里绕出一条弯路。第二次，我要求工具中心沿直线接近，七个关节便以不同节奏配合。第三次，我让工具中心沿夹具边缘绕过一段圆弧，手的几何承诺又变了。最后一次，几何路线完全不变，我只把起步和停下变得柔和，腕部安装座的晃动便小了一些。

我问：“明明从同一个地方去同一个地方，为什么这四次还不是同一个动作？”

博士这才给前三种几何承诺命名：只约定各关节终点的叫点到点运动（PTP），约定工具中心走直线的叫直线运动（LIN），约定它沿圆弧经过中间点的叫圆弧运动（CIRC）。第四次改变的不是路线，而是沿路前进的节奏。

我想起第三幕里规划落脚与摆腿时，空间上的曲线和“何时走到哪里”始终是两层问题。那套区分也许可以用于手臂，但现在七个关节的速度、加速度和更突然的变化都要逐一检查。

先决定经过哪里，还是先决定怎样快慢

博士把不带时间的几何曲线称为路径，把沿路径前进的时钟称为时间参数化，把二者合成的带时间状态称为轨迹。一段动作至少包含任务承诺、几何路径、时间律和真实执行四层含义。

几何路径用参数 s∈[0,1]s\in[0,1]s∈[0,1] 表示：

q=q(s).q=q(s).q=q(s).

它只回答关节配置按什么形状从起点连到终点，不回答历时多久。

时间律为

s=s(t),s(0)=0,s(T)=1,s=s(t), \qquad s(0)=0,\quad s(T)=1,s=s(t),s(0)=0,s(T)=1,

通常还要求

s˙(t)≥0,\dot s(t)\ge0,s˙(t)≥0,

保证沿同一路径向前推进。

二者合成时间轨迹：

q(t)=q(s(t)).q(t)=q(s(t)).q(t)=q(s(t)).

轨迹是期望状态随时间的记录；真实执行还需要闭环反馈，把实际 q,q˙q,\dot qq,q˙​ 与参考比较，并在偏差或环境变化时停下或重算。路径、轨迹和执行不是三个同义词。

本章形成的自由空间运动计划仍是候选。它还要在下一章接受整臂碰撞检查，真实身体能否跟上也要由执行反馈另行判断。

关节空间直线与 TCP 直线保留的是不同承诺

在本章采用的代表性定义里，PTP 先在关节空间组织同步的点到点变化。关节空间线性路径为

q(s)=q0+s(q1−q0).q(s)=q_0+s(q_1-q_0).q(s)=q0​+s(q1​−q0​).

它让每个关节随 sss 线性变化，但 TCP 经过的工作空间曲线通常不是直线。

PTP 不保证工具中心走直线。LIN 则把工具中心的位置约束为笛卡尔直线，例如

p(s)=(1−s)p0+sp1,p(s)=(1-s)p_0+sp_1,p(s)=(1−s)p0​+sp1​,

姿态则通过 SO(3)SO(3)SO(3) 上的一致插值生成。CIRC 把直线位置约束换成由起点、中间点和终点确定的圆弧约束。两者都需要沿途求逆运动学，也都会引出新的边界：

  • 相邻采样点可能落到不同 IK 分支，关节发生跳变；
  • 中间位姿可能不可达，尽管两端可达；
  • 路径可能穿过奇异位置；
  • TCP 直线并不说明手肘、前臂或夹爪外形走直线；
  • 逐点 IK 通过不证明点与点之间连续。

因此，要先说明任务真正需要保持什么。若只需在开阔空间换姿态，关节空间路径可能更自然；若工具必须沿直线接近或绕固定边缘，LIN/CIRC 的笛卡尔约束才可能是硬要求，但要连同 IK 分支与整臂几何一起处理。PTP、LIN、CIRC 描述的是几何责任，不是三个由低到高的安全等级。

梯形速度和 S 曲线改变的是沿路节奏

几何路径冻结以后，还要选择 s(t)s(t)s(t)。梯形速度让路径速度先匀加速、再保持、最后匀减速；画出来像一座平顶的山。它容易解释，却在理想分段处让加速度突然改变，意味着加加速度出现尖峰。

S 曲线把加速度也分段缓慢升起和放下，以加加速度边界换取更柔和的起停。它不改变几何路径，只改变沿路怎样起步、巡航和停下。

还有一种选择不是直接给加加速度设上限，而是在声明的端点条件下，让整段运动的加加速度平方积分尽量小：

min⁡q∫0T∥q...(t)∥2 dt.\min_q\int_0^T \left\|\dddot q(t)\right\|^2\,dt.qmin​∫0T​∥q...​​(t)∥2dt.

博士把它称为 minimum jerk。在只给定起终位置、时间，并令端点速度和加速度为零的点到点问题中，它会得到后文的五次曲线。这个想法最早是为解释人的手臂运动而提出的；迁移到机器身体时，它只是一种平滑性准则，不是“像人一样”或普遍最优的证明。

多项式分段与样条不是“把点连起来”这么简单

多个途经点逐段使用多项式时，要在连接点说明连续到哪一阶：C0C^0C0 只保证位置相接，C1C^1C1 再保证速度连续，C2C^2C2 还保证加速度连续。若只有 C0C^0C0，延长时间不能消除几何方向突变，只能让身体在尖角处停住再走。

博士把由控制点、次数和节点向量定义的分段多项式称为 B 样条。次数为 ppp 且内部节点不重复时通常具有 Cp−1C^{p-1}Cp−1 连续性，重复节点会降低连续阶次。控制点通常不是必经点。

所以“使用三次样条”还必须回答：端点导数是什么，是否经过途经点，节点和分段时间怎样分配，以及限位与采样点之间的峰值怎样检查。

固定路径怎样重新安排沿路速度——TOPP

若几何路径 q(s)q(s)q(s) 已经冻结，时间参数化问题可以在不改变路线的条件下寻找 s˙,s¨\dot s,\ddot ss˙,s¨。

关节速度约束

∣q˙i∣=∣qs,i(s)s˙∣≤vˉi|\dot q_i| = |q_{s,i}(s)\dot s| \le\bar v_i∣q˙​i​∣=∣qs,i​(s)s˙∣≤vˉi​

给出每个 sss 处的路径速度上限。

关节加速度约束

−aˉi≤qss,i(s)s˙2+qs,i(s)s¨≤aˉi-\bar a_i \le q_{ss,i}(s)\dot s^2 +q_{s,i}(s)\ddot s \le \bar a_i−aˉi​≤qss,i​(s)s˙2+qs,i​(s)s¨≤aˉi​

给出路径加速度的可行区间。

博士把在固定路径上寻找满足约束的时间律称为时间最优路径参数化（TOPP）。常见实现先离散路径，再向前传播从起点可达的速度集合，向后传播能够在终点条件下停住的集合；两者交集为空时，当前路径与边界下不可行。

同一责任还有几种常见实现边界：

  • TOTG：在固定且足够光滑的路径上，依据速度和加速度边界寻找沿路时间；它不负责重新规划绕障路线。
  • TOPP-RA：把每个离散路径位置上的可达、可控速度集合向前和向后传播，交集决定能否继续。
  • 在线限加加速度轨迹生成：不是从静止重新处理整条路径，而是从当前的位置、速度和加速度出发，生成到目标状态的受限轨迹。

Ruckig 是第三类方法的论文与实现实例。本章只借它辨认输入、输出和边界，不表示这具身体已经接入该软件。

“不可行”可能来自路径不光滑、起终速度冲突、边界过紧，也可能来自离散过粗或数值容差不合适。这个结论本身还没有指出应该换路、重定时还是改进数值检查。

TOPP 不会绕开障碍、改变途经点或修复 IK 分支跳变。若问题来自路径尖角或几何冲突，就要换路径或增加平滑段，而不只是继续放慢。

Hung Pham 与 Quang-Cuong Pham 的基于可达性分析的 TOPP 论文给出前后两遍传播的一种系统形式；本章只迁移固定路径与时间律分离、可达集合和不可行分支。

采样点通过，峰值仍可能藏在点之间

若只在 t=0,0.1,0.2,\ldots 检查约束，窄小的速度、加速度或加加速度峰值可能恰好落在两个时刻之间；路径曲率大的区间也可能被稀疏 sss 采样漏掉。

可采用：

  • 对多项式导数求极值根并检查端点；
  • 使用区间界、凸包性质或解析上界；
  • 在高曲率、近限位和约束余量小处自适应细分；
  • 把连续保证与仅离散抽查明确区分。

采样加密仍不等于数学上的连续证明。若没有连续界，就应保存最大采样间隔和未覆盖风险。

自由空间运动计划怎样保存路径与时间

输入：
  当前逆运动学评估中的起点与终点
  几何运动模式及其约束（PTP / LIN / CIRC / 自定义路径）
  关节位置、速度、加速度与加加速度边界
  端点导数条件
  路径与插值方式
  身体、标定、约束版本与最晚完成时刻

1. 核对起点与终点的身体、标定、坐标系和有效期。
2. 构造 q(s)：
   - 说明关节空间或笛卡尔空间；
   - 检查连续性、IK 分支、端点和关节限位。
3. 计算 q_s、q_ss，以及需要时的 q_sss。
4. 构造 s(t)：
   - 多项式/样条候选，或 TOPP 前后传播；
   - 保存起终速度、加速度和不可行原因。
5. 检查每个关节的峰值、单位、峰值时刻和余量。
6. 对小余量区间细分或使用连续界。
7. 任何非有限值、导数不存在或边界失败都停止发布。

输出初版自由空间运动计划：
  几何运动模式及其任务承诺
  路径表示与参数范围
  时间律与总时长
  导数连续性与端点条件
  各关节峰值及其位置
  采样或连续检查方法
  身体、标定、约束版本与执行期限
  留待后续 = [自碰/环境碰撞、跟踪、动力学、接触]

本章只使用运动学导数边界。完整动力学时间最优还需要质量、惯量、驱动力矩与负载模型，不能用 TOPP 名字提前代替。

时间让姿态成为过程，却还没有让过程避开世界

我终于能说明手臂在每个时刻应该处于哪里，也能区分“路线必须改变”与“同一路线需要放慢”。端点不再替中途作证，采样点也不再替连续曲线作证。

可当这条轨迹进入狭窄工具架时，屏幕只画出了 TCP 的线。TCP 离架子很远，手肘却从另一侧擦过边缘。下一章必须让整条身体，而不只是手的一个点，进入对世界的几何判断。

练习 2：为什么在中间途经点必须停一下

两段关节空间直线只保证在途经点位置相接，进入方向与离开方向不同。程序为两段各分配更长时间，却仍在途经点报告速度方向跳变。

判断单纯继续延长时间能否在非零速度下修复。针对途经点必须精确经过和不必精确经过两种情况，分别提出处理办法。

练习 3：LIN 已经走直线，腕关节为什么突然跳了一圈

笛卡尔插值的每个采样位姿都能求出 IK，但第 17、18 个样本来自不同腕部解支，关节差接近 2π2\pi2π。角度数组直接相减后仍在关节范围表示内。

决定应增加时间、做角度主值，还是回到连续 IK 分支与路径定义。说明仅修主值为什么仍不足。

练习 4：TOPP、TOTG 报告失败时应该换时间还是换路

候选 A 的几何路径光滑，TOPP 因当前时长猜测过短而被程序标成失败；候选 B 在途经点有尖角，qsq_sqs​ 左右极限不同，却要求以非零速度通过；候选 C 在某段关节位置已经越限。

分别判断应重做时间传播、增加平滑分段或直接更换路径。说明 TOPP 能修复和不能修复的边界。
