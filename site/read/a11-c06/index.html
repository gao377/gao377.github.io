第十一幕

数据已经很多，为什么关键失败还没有出现——任务域、覆盖与尾部空洞

数据集登记很快变得热闹起来。同一次平稳试转能产生几千帧，连续相机流让“样本数”迅速上涨。可我把人员进入、低摩擦、通信断链和保护停止放在一起查询，结果仍是零。

“这么多数据，为什么没有我真正担心的那一种？”我问。

博士说，数量回答“保存了多少”，coverage 回答“世界的哪些部分留下了独立证据”。

先写清机器人被允许面对的世界

operational design domain（ODD） 不是一行“城市环境”，而是一组会改变任务意义和风险的条件。第十一幕的 coverage key 至少包括：

身体与配置
任务阶段、资产版本与地图版本
天气、表面与能见度
状态、动作与接触情形
传感器健康、延迟与通信状态
人员是否在场以及接管状态
失败、恢复与结果
证据等级与谱系组

四足、轮足、带操作部件的轮足和当前人形记录要按当时配置分层。轮足在湿路转弯的数量不能填补人形在湿楼梯上的格子；我的记忆与任务连续，不等于不同身体的状态空间可以互换。

单轴覆盖会隐藏联合空洞

也许档案里有低摩擦，有人员进入，也有断链；若它们从未在同一相关任务阶段共同出现，联合格子仍为空。设条件集合为 z=(b,t,w,p,c)z=(b,t,w,p,c)z=(b,t,w,p,c)，真正要问的是：

Ngroup(z)=#{independent Episode groups compatible with z}.(1)N_{group}(z)=\#\{\text{independent Episode groups compatible with }z\}. \tag{1}Ngroup​(z)=#{independent Episode groups compatible with z}.(1)

这里数的是独立 Episode group，不是相邻帧。一个 run 的一万帧不能当一万个独立失败机会；复制、重采样、同源派生也不增加独立覆盖。

连续变量不必粗暴切成无穷格子，可以保存已观察 support、边界附近密度和离目标 ODD 的距离。离散变量则保留 nominal / boundary / failure / recovery / unknown / not-attempted。zero 还要分清：从未尝试、尝试但未发生、发生却未记录，不是同一件事。

尾部事件不能被平均指标淹没

保护停止未触发、人员安全边界越过、错误身体收到动作等事件即使罕见，也可能是 zero-tolerance event。它们要单独登记，不能用名义成功帧的比例冲淡。

覆盖流程可以写成：

计数前先声明任务域、任务、身体和失败的必要组合
按因果关系和谱系身份组织记录
标出已观察支持、边界、失败、恢复、未知和尚未尝试
分别计算单项与联合覆盖，不填补空白格
标记近重复记录和真正独立的经历组
提出带许可、能量和安全约束的补证方案
绝不自动执行采集建议

博士把当前数据集固定为 40 个 current Episode group，并要求按 24/8/8 拆分；另有 12 个独立 regression group。这些数量只定义眼前的分组、拆分和空洞问题，不能生成虚假的覆盖百分比。

空洞是一项知识，不是一张采集命令

若“低摩擦 + 人员进入 + 断链”为空，最诚实的状态是 coverage=ABSENT、相关结论 UNKNOWN。补证顺序可以先从已有失败回放、经许可仿真、HIL、隔离低能量试验开始；任何真实人员、湿地或断链组合都要另行风险评估和许可。

有些条件不应为了填表而主动制造。知识的边界被看见，本身就是 coverage map 的价值。

练习

练习 1：一万个相邻帧

同一名义 run 保存一万个相邻图像帧，另有五个独立保护停止 Episode。比较它们对“视觉纹理多样性”和“保护停止因果路径”两种 coverage 的意义。

练习 2：每个单项都有，组合为零

档案分别包含低摩擦、人员进入和网络断链，但从未在当前人形执行泵站任务时共同出现。判断可以写出的结论和不可写的结论。

练习 3：怎样补一个危险空洞

团队希望直接在湿地、人员靠近且网络受限时让身体重复任务。给出一条更有边界的补证路线，并说明什么情况下仍应保持 unknown。

参考答案

空洞显露出来以后，我重新对齐一段旧数据，生成了新 revision。可旧 calibration 随后被撤销：哪些数据集、拆分和候选模型也该随之失效？只靠“最新版”文件名已经答不出来。
