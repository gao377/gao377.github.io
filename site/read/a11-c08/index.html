最相似的记录，为什么未必是同一段过去——记忆索引、检索与矛盾保留

我用“泵站、潮湿、动作已施加、设备未变化”去搜索档案。第一条结果几乎逐字相同，地点却在另一座设施，body revision 也属于轮足时期；一条排名很低的短记录只有 action ID 和设备号，却与当前冲突档案指向同一件动作。

“最像我的回忆，为什么不一定是我的那段过去？”

博士说，语义相似只回答文字或表示有多接近。记忆还要回答时间、地点、身体、对象、事件和来源是否相容。

记忆索引不是一个相似度列表

记忆索引同时维护多路入口：

  • temporal：事件时间、时钟域、epoch 和不确定度；
  • spatial：地点、地图、frame 与资产位置；
  • identity：task、body/config、object、participant、action/Event ID；
  • event/failure：接触、接管、失败、恢复和 outcome；
  • semantic：文本、图像或状态表示的相似候选；
  • provenance：raw/derived、transform、source、duplicate family 与 evidence。

检索次序是先用当前问题不可妥协的身份做结构化过滤，再在兼容候选里排序：

C={r∣F(r,q)=compatible or explicitly unknown},rank⁡(r)=∑kwksk(r,q).(1)\mathcal C=\{r\mid F(r,q)=\text{compatible or explicitly unknown}\}, \qquad \operatorname{rank}(r)=\sum_k w_k s_k(r,q). \tag{1}C={r∣F(r,q)=compatible or explicitly unknown},rank(r)=k∑​wk​sk​(r,q).(1)

这里 unknown 候选可以被保留供调查，但不能和 confirmed compatible 混成一类。权重、query revision、filter 和 score components 都要保存；否则同一问题以后无法复现为什么返回了这组记忆。

第一名只是一条候选，不是当前事实

检索结果至少返回 top-k 的不同来源，而不是十份相同抄本。每条结果同时附带：

原始或派生身份及其谱系
时间、地点、任务、身体、对象和动作是否兼容
证据等级以及缺失或未知字段
语义相似与结构化相似各自的分数
重复记录族与冲突集
当前版本、任务域和权限是否兼容

高语义相似但身体和任务不同的记录，可以帮助提出假设，不能直接复用动作。低语义相似但 action ID 相同的记录，可能更适合做身份 reconciliation。来源复制的多条结果要折叠成一个家族，同时保留各派生解释。

矛盾也是记忆的一部分

传统检索常把“最佳答案”返回给当前状态。这里不能这样做：若两份材料对同一 outcome 冲突，memory index 返回 contradiction set、各自来源和 resolution status。它不替 conflict set 投票，也不把第一名写进地图、计划或控制器。

legacy-fragment-07 是一个特意留下的边界。它与若干当前词句相似，派生时间看似来自未来，却缺 raw actor、clock domain 和 body revision。memory index 可以说“找到了这条候选”，不能说“未来的我留下了它”，更不能借它裁定博士、我或人类与机器的关系。

这不是把哲学问题赶出技术，而是让技术诚实地说明：档案保存的是被选择、被记录、被转换的痕迹，不是过去本身。主体连续性也不能由 embedding 距离证明。
