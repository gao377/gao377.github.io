第十一幕

“去旧泵房旁边”究竟指哪里——开放词汇导航、具身记忆与社会约束

一位工人对我说：“去旧泵房旁边等我。”地图和记忆给出三个候选：旧建筑本体、仍挂着“旧泵房”门牌的新仓间，以及一台已经拆除的旧泵曾经所在的位置。最近路线穿过有人检修的窄道。

我可以识别每个词，却仍不知道“旁边”属于哪个对象、从谁的视角说、现在还能不能去。

博士把“词语究竟指向哪个现实对象”称为指称消解；开放词汇与 VLN 可以提出候选，具身记忆提供历史语境，社会导航则继续检查人的活动、许可与礼让边界。

开放词汇只扩大候选，不制造唯一指称

开放词汇视觉/检索把图像区域或地图实体与自然语言描述放入可比较表示，能在固定类别表之外提出对象候选。但相似度并不解决 ontology mismatch：

  • “旧”可能描述年代、用途、俗称或相对新建筑；
  • “泵房”可能是建筑、门牌、设备区域或人的共同记忆；
  • 已拆除实体仍可存在于历史 memory，却不应作为当前可到达目标；
  • unknown class 与低置信检测不能被最相似标签强行命名。

每个 referent candidate 要带 entity ID、map revision、来源、时间、score、contradiction 和 current existence status。

“旁边”需要一个关系 frame

关系不是纯距离。可把候选写作

rj=(ej, Rbeside, Fspeaker/context, t, b),(1)r_j=(e_j,\,R_{beside},\,F_{speaker/context},\,t,\,b), \tag{1}rj​=(ej​,Rbeside​,Fspeaker/context​,t,b),(1)

其中 eje_jej​ 是实体，RRR 是关系，FFF 是说话者/场景采用的参考 frame，ttt 是当前时间，bbb 是身体。人可以从门口视角把左侧等待区称为“旁边”；对当前人形，狭窄缝隙即使几何很近，也可能不可达或侵犯工作边界。

指称消解 因此结合 speaker、对话历史、手势/视线、任务、地图版本与具身可达性。存在多个仍可行动的解释时，最可靠的动作是询问，而不是把语言模型的自信当许可。

视觉—语言导航提议子目标，社会导航限制怎样到达

VLN 使用观察—语言—动作历史提出 referent、subgoal 或 route candidate。这里仍保持模块化责任：

语言、说话者与上下文
→ 带来源和版本的指称候选
→ 语义与地图落点
→ 考虑身体的可达性和路线候选
→ 权限、工作区域和社会约束
→ 影子子目标或澄清请求

社会导航关注动态人员、队列、个人空间、工作区、通行方向、礼让和明确许可。最近路线穿过有人工作的窄道时，即使碰撞几何无冲突，也可能不应通行。当前人形适配人类尺度，让我更容易使用门、通道和人的工具，却不自动赋予人的意图、身份或权利。

具身记忆保存的是谁在何时看见什么

memory index 中“旧泵曾在这里”的条目可以帮助理解称谓，却必须保留 place/event/actor/body/time/revision。拆除前记忆不应覆盖当前地图；当前地图也不必删除人的历史叫法。两者并列，正好解释为什么一句普通话会有多个合理入口。

按语言、地点和记忆检索彼此不同的指称候选
对当前动作排除已撤销或不存在的目标，但不删除历史记录
绑定关系坐标、说话者以及地图和实体版本
检查身体可达性、工作区域权限和社会约束

如果仍有多个可行动的指称：
    停下并提出能够区分它们的澄清问题
否则：
    生成带来源和后备路径的影子子目标
不把旧记忆直接映射成关节或力矩控制

练习

练习 1：三个“旧泵房”候选

建筑、门牌和已拆泵位置都与语言相似，说话者仍在场。说明先排除什么、先问什么，以及怎样让问题一次区分多个候选。

练习 2：最近路线穿过工作窄道

路线几何无碰撞，但检修人员占用通道，permission 未知。判断是否通行，社会约束与几何约束分别记录在哪里。

练习 3：过期记忆是否要删除

旧泵已经拆除，人的俗称仍沿用。说明当前地图和 embodied memory 应如何同时保存，避免“过期”变成“从未存在”。
