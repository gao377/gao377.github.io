多次查询怎样共用一张自由空间路线图——PRM

概率路线图（PRM）通常分为两个阶段。

建图阶段：

当节点或采样预算尚未耗尽：
  在关节域中采样 q
  若 q 无效，则舍弃
  把 q 加入路线图节点
  寻找 k 个最近邻居，或半径内的邻居
  只有整段局部连接有效，才加入规划边

查询阶段：

验证起点与终点
分别连接到路线图中的有效邻居
用图搜索寻找节点序列
按当前版本重新验证返回的全部规划边

静态场景中有许多起终查询时，多次查询可以分摊建图成本。若工具架移动、身体或标定版本改变，受影响节点和边必须失效或重查；旧路线图不会因名字里有“概率”就自动适应新世界。

窄通道的困难在于通道体积很小，均匀样本很少落入。增加样本、使用边界/桥接等有依据的采样策略，或保留全域支持的混合分布可以改善机会；只在一个启发区域采样则可能永久看不到另一条路。

Kavraki 等人 1996 年的 PRM 原始论文系统提出了在高维配置空间中构建随机路线图并复用查询的框架。

一棵树怎样朝尚未探索的区域生长——RRT

RRT 从起点开始：

树 T = {起点}
预算尚未用尽时：
  q_rand = 随机采样()
  q_near = 树中距离 q_rand 最近的节点
  q_new  = 从 q_near 向 q_rand 局部推进一步
  若新节点与连接它的规划边都有效：
      把 q_new 与父规划边加入树 T
  若终点可以通过有效规划边连接：
      返回候选路径

Voronoi 区域大的节点更可能成为随机样本的最近节点，所以树倾向于伸向尚未覆盖的区域。它不是显式计算 Voronoi 图，也不保证第一条路径短。

原始 RRT 适合单次查询和快速探索，但路径常有折线、绕行和净空过小的问题。它返回首次候选后仍需后处理与复核。

两棵树为什么常更快得到第一条路——RRT-Connect

RRT-Connect 同时从起点与终点生长两棵树：

  1. 一棵树向随机样本延伸一步；
  2. 另一棵树朝新节点反复连接；
  3. 若两树通过有效规划边相遇，拼接两条父链；
  4. 每轮交换两棵树的角色。

从两端同时进入狭窄区域，可能比单树等到抵达目标更快。这里的“连接”不是跳过碰撞检查：反复前进的每一小段和最终拼接边都必须检查。

它也不保证第一条路径质量。快速找到上方绕行，不代表侧方路径不存在或更差。

Kuffner 与 LaValle 的 RRT-Connect 原始会议论文描述了这种双向快速连接策略。

“概率完备”不是“这次一定找到”

在自由空间存在具有正净空的可行路径、采样分布覆盖相关区域、局部连接与碰撞判断正确等条件下，某些 PRM/RRT 系列算法随着样本或时间趋于无穷，漏掉解的概率趋向零。博士把这类性质称为概率完备性。

它与以下说法不同：

  • 有限预算内保证成功；
  • 超时就证明无解；
  • 净空为零的路径也满足同样条件；
  • 任意固定步长、采样偏置与近邻实现都保留证明；
  • 现实几何与模型完全一致。

因此，终止状态必须区分：

  • 找到候选：找到并复核了一条当前版本路径；
  • 超时：预算用尽，尚未找到；
  • 样本拒绝过多：过多样本落在无效区域；
  • 起点/终点无效：端点本身无效；
  • 碰撞查询未知：边检查没有得到合格结论；
  • 资源失败：内存、非有限值或其他实现失败。

这些状态都不是“已经证明无解”。通用高维采样器通常不能由一次失败给出无路证明。

后处理会创造新规划边，必须重新检查

采样路径常包含许多折点。捷径化会随机选择两个非相邻节点；若局部连接有效，就用一条直边替换中间片段。样条平滑则改变整段几何。

每次改变都可能：

  • 穿过原路径绕开的障碍；
  • 降低净空要求；
  • 越过关节限位；
  • 进入奇异区域；
  • 使路径导数或时间参数化改变。
