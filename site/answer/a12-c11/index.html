第十二幕·参考答案：两副身体抬着一个物体，为什么各自正确仍会互相用力

练习 1

先停止增加作用和速度，保留仍可验证的对象支撑；核 object/contact/frame/clock revision、两端增益、接触模式和消息 freshness。若内部力无法在约束内降低，就执行已验证的同步减载/共同卸载；卸载合同不成立时各端进入独立保护并停止。

这里要分开对象所受净力旋量与零空间中的内部力。对象运动正确，只说明 GfGfGf 符合目标，不能保证两端没有在内部互相顶住；回代 GfGfGf 与 GNGλ=0GN_G\lambda=0GNG​λ=0，同时查看每一端作用，才能看清分配。35 N 只是给定诊断值，不是现实通用门限。

练习 2

先拒绝拼接两条 wrench，保留各自 raw 和 frame identity；用正确的 wrench transform、接触点和排列把两端变到同一 object frame，再做 power/sign 与对象净作用回代。

这是抓取矩阵的坐标系语义。数组长度相同，不代表两条力旋量已经在同一参考系中；用一个已知纯力案例检查转换后的力矩方向和单位，可以发现符号或排列错误。题目不证明传感标定正确。

练习 3

先拒绝要求运输设备穿窄门，暂停对象轨迹；重新选择道路侧可达的放置点、由我的人形身体在允许区域继续，或整体卸载并重分配。cooperative MPC 只能在给定身体可行域内优化。

对象任务与每副身体自己的可达域必须同时满足。联合控制不能把运输设备过不了窄门这一事实平均掉；应把候选轨迹分别放回人形和运输设备的地图、身体包络与停止合同检查。题目没有保证一定存在替代轨迹。
